## 1. 분량 설계 (20~25분 기준 확정)

```
선택지 3개 기준, 다이아몬드 분기 구조

시간 배분
오프닝     2~3분
스테이지1  4분
스테이지2  4분
스테이지3  4분
최종관문   4~5분
엔딩       2~3분
─────────
총합      20~23분
```

### 텍스트박스 역산

```
1회 플레이 체감 분량
오프닝         6~8박스
스테이지1~3    각 12~16박스 × 3 = 36~48박스
   (분기별 텍스트 최대한 공유, 다이아몬드 구조 적극 활용)
최종관문       22~28박스
엔딩 (1개 도달) 6~10박스
─────────────────
1회 플레이 체감     70~94박스

전체 제작 분량 (모든 분기 포함, 작가가 실제로 써야 하는 총량)
오프닝         6~8박스
스테이지1~3    각 18~24박스 × 3 = 54~72박스
최종관문       22~28박스
엔딩 3종       18~28박스
히든 엔딩      4~6박스
─────────────────
총 제작 분량    104~142박스
```

### 가이드라인

```
vn_script.md 집필 시 이 범위(104~142박스)를 한참 초과하면 분량 과다 신호
한참 못 미치면 분량 부족 신호

씬당 텍스트박스 = 대사/지문 1박스당 평균 60~100자 (타이핑 애니메이션 고려)

다이아몬드 구조 원칙:
스테이지1~3은 선택에 따라 스탯 변화는 다르지만 중간 지점에서 합류
최종관문에서만 분기별로 완전히 다른 텍스트 작성 (진짜 갈림)
→ 선택지마다 완전히 다른 장면을 쓰지 않는다 (집필량 폭증 방지)
```

## 2. 배드엔딩 후 처리

### 사망 즉시 흐름

```
1. 화면 전환: 랜턴이 꺼지는 연출 (1~2초)
2. 오르페아 사망 코멘트 (필수, 사망 원인별 분기, 1~2박스)
3. 선택 화면 제시: [마지막 스테이지부터 재도전] / [처음부터] / [그만두기]
```

### 사망 코멘트 — 원인별 패턴 (예시, 에피소드마다 커스텀)

```
지뢰 적중형: "신앙심 부족이군요. 흔한 결말입니다."
도주 실패형: "도망이라... 역사에 흔적도 안 남았겠군요."
스탯 고갈형: "체력이 먼저였나요. 안타깝지만 흔한 계산 착오입니다."
→ 사망 원인마다 짧은 코멘트 세트 2~3개 패턴 필요
```

### 재시작 지점

```
기본 원칙: 마지막 스테이지부터 (처음부터 X)
이유: 완전 처음부터 재시작은 이탈 유발 (반복 읽기 피로)
      마지막 스테이지부터면 "이번엔 다르게" 동기 유지

예외: 로그라이크 포맷
      "처음부터"가 핵심 메커닉이므로 이 규칙 미적용
      (죽을수록 정보 축적이 재미 포인트)
```

### 재도전 시 스킵 처리

```
이미 읽은 텍스트(오프닝~직전 스테이지) → 자동 스킵 또는 스킵 버튼
선택지 부분 → 스킵 안 함 (다시 선택해야 함)
직전 사망 스테이지 도입부 → 새로 읽되 "스킵 가능" 표시

기술 구현:
세이브 데이터에 "읽음 처리된 텍스트박스 ID" 저장
동일 박스 재진입 시 빠른 재생 옵션 제공
```

## 3. 스프라이트 표정 트리거

### 레이어 구조

```
레이어 1 — 공통 트리거 카테고리 (모든 캐릭터 공유, 상황 분류)
레이어 2 — 캐릭터별 표정/반응 매핑 (캐릭터마다 다름, 톤 반영)

⚠️ 캐릭터마다 표정 시트를 그대로 복붙하면 안 됨.
   트리거 "상황"은 공통, 그 상황에 대한 "반응"은 캐릭터 고유.
```

### 레이어 1 — 공통 트리거 카테고리 (9종)

```
1. 기본 (디폴트, 트리거 없을 때)
2. 위험한 선택 직후 (플레이어가 위험한 선택을 했을 때)
3. 예상 밖 선택 직후 (히든 트리거 포함)
4. 명백한 실수 직후 (사망 직결 선택)
5. 핵심 정보 전달 (지뢰 경고, 중요 단서)
6. 대기/반복 (선택 고민 오래 끌 때, 같은 패턴 반복)
7. 사망 직후 코멘트
8. 불완전 성공 (일반 생존 엔딩 등)
9. 침묵 반응 (플레이어가 선택하지 않고 대기)
```

### 레이어 2 — 캐릭터별 매핑

**오르페아 (확정 — 9종 표정 시트 보유)**
```
기본 → 기본미소
위험한 선택 → 냉소적미소 ("유감이군요" "역사적으로 인기있는" 류)
예상 밖 선택 → 흥미 ("흥미롭군요" "예상 밖" 류)
명백한 실수 → 독설 ("통계적으로" 류)
핵심 정보 전달 → 진지
대기/반복 → 지루함
사망 직후 → 조롱
불완전 성공 → 실망 ("운이 좋으셨습니다" 류)
침묵 반응 → 관찰자모드
```

**네피아 (⚠️ 전용 표정 시트 미제작 — 일러스트 작업 시 함께 설계 필요)**
```
방향성만 확정:
위험한 선택 → 더 부드러운 미소 (오르페아와 반대로 친절해짐)
사망 직후 → 표정 변화 거의 없음 (조롱 아님, "무표정/평온")
전반적으로 오르페아보다 표정 변화 폭이 작음 (이미 결론 낸 자의 정적임)
```

**역병의사 (⚠️ 표정 시스템 자체 적용 불가 — 별도 체계 필요)**
```
얼굴 없음 → 9종 표정 매핑 불가능
대신 "가면 상태" 트리거로 대체:
- 가면 기울기 (정상/기울어짐)
- 망토 흔들림 (정지/흔들림)
- 데이터 노이즈 강도 (없음/약함/강함)
→ 별도 설계 필요, 일러스트 작업 시 함께
```

**글리치 NPC (트리거 무시가 컨셉)**
```
같은 트리거에도 표정이 예측 불가하게 깨짐
"정상 작동 안 함"이 캐릭터성
→ 정교한 매핑 불필요, 오히려 불규칙성을 의도적으로 설계
```

### 에피소드 핵심 NPC (플레이어와 직접 대적하는 1~2명)

```
표정 세트: 3~4종으로 축소 (제작 효율)
- 중립 (기본)
- 적대적 (분노/의심/위협)
- 동요 (불안/긴장/슬픔)
- (선택) 4번째: 역할에 따라 호의적 or 승리감/조소

트리거 (압축 버전):
중립 → 평상시 디폴트
적대적 → 플레이어 의심/위협 대사
동요 → NPC 본인 위기 또는 거짓말 시
4번째 → 결정적 순간 (배신/도움)
```

### 나머지 단역 NPC

```
표정 변화 없음, 고정 이미지 1장
감정 표현은 텍스트/배경색/화면 효과로 대체
(예: 적대적 분위기일 때 화면 빨간 비네팅 효과)
```

### vn_script.md 작성 규칙

```
대사에 표정 태그 인라인 명시:
[오르페아: 냉소적미소] "유감이군요. 신앙심이 부족했나 봅니다."
[오르페아: 흥미] "...흥미롭군요. 예상 밖의 선택입니다."

제약:
- 한 씬에서 표정 전환 최대 2~3회로 제한 (산만함/로딩 부담 방지)
- 선택지 직전엔 항상 "기본" 또는 "진지"로 리셋 (결과 스포일러 방지)
```

## 4. BGM 전환 기준

### 레이어 구조

```
레이어 1 — 에피소드 테마곡 (기본 베이스, 오프닝~엔딩 까는 기본 분위기)
레이어 2 — 긴장도에 따른 변주 (같은 곡의 편곡 버전)
  평온 → 기본 버전 (현악 위주, 느림)
  긴장 → 타격감 추가 버전 (퍼커션 추가)
  위기 → 풀 편곡 (전체 악기, 빠른 템포)

전환 트리거: 스탯 종합 위험도 임계값 기준 자동 전환
  생존율 50% 이하 → 긴장 버전
  생존율 25% 이하 → 위기 버전
```

### 명확한 BGM 교체 지점

```
1. 오프닝 → 스테이지1 진입
   인트로용 짧은 테마 → 본편 테마

2. 고정 캐릭터 등장 시
   오르페아 등장 = 오르페아 테마 (랜턴 모티프, 따뜻한 음색)
   네피아 등장 = 네피아 테마 (오르페아 테마의 반전/마이너 버전)
   역병의사 등장 = 별도 unsettling 테마 (낮은 음역대, 불규칙)

3. 선택지 제시 순간
   짧은 정적 또는 음악 볼륨 다운 (긴장감 조성)

4. 엔딩 진입
   퍼펙트 → 테마곡 메이저 버전
   일반생존 → 테마곡 기본 버전 유지
   사망 → 테마곡 마이너/단절 버전 (랜턴 꺼지는 연출과 동기화)
```

### 트랙 제작 수량 (Suno/Udio 기준)

```
에피소드 고유 (매 에피소드 신규 제작): 2곡
  1. 에피소드 기본 테마 (평온 버전)
  2. 긴장 버전 (같은 테마 편곡)

공통 재사용 (최초 1회만 제작, 이후 전 에피소드 재사용): 6곡
  3. 오르페아 테마
  4. 네피아 테마
  5. 역병의사 테마
  6~8. 엔딩 테마 3종 (퍼펙트/일반/사망)

→ 첫 에피소드(세일럼)에서 공통 6곡 + 고유 2곡 = 총 8곡 제작
→ 이후 에피소드는 고유 2곡만 추가하면 됨
```

## 5. EP.01 온보딩 — 확정: 설명 없이 바로 던지기 (방식 C)

```
결정: 튜토리얼/설명 없음. 시스템(스탯/선택지/오르페아 정체)을
      플레이어가 직접 만지면서 파악하게 함

근거:
- 오르페아의 냉소적 캐릭터성과 "친절한 튜토리얼"은 톤이 충돌
- 요즘 유저는 UI를 직관적으로 파악하는 능력이 충분함
- 불필요한 설명은 몰입을 깸
- 닥치면 하게 되어 있음

적용:
EP.01부터 모든 에피소드 공통 — 설명 단계 없이 바로 오프닝 진입
오르페아의 정체, 스탯의 의미 등은 플레이 경험과 대사를 통해
자연스럽게 체득되도록 vn_script.md 집필
```

## 6. 리플레이 설계 (기본 생존형 기준 — 확정)

```
전제: 배드엔딩은 최종관문에서 발생
      (오프닝~스테이지3은 이미 클리어한 상태)

재도전 시작점: 최종관문 도입부

이미 읽은 구간 처리 (오프닝~스테이지3):
빠른 스킵 — 텍스트 즉시 표시, 타이핑 애니메이션 생략

최종관문 도입부:
새로 읽되 "스킵 가능" 표시 제공
(같은 도입부를 여러 번 사망 후 재도전 시 빠르게 넘길 수 있도록)

선택지 A/B/C:
항상 다시 제시 (자동 선택 옵션 없음)
사망 원인이 된 선택이므로 플레이어가 직접 다시 골라야 함
→ "이전과 같은 선택" 자동화 버튼은 불필요 — 제외

기술 구현:
세이브 데이터에 "읽음 처리된 텍스트박스 ID" 저장
동일 박스 재진입 시 자동 빠른 표시

⚠️ 예외: 로그라이크 포맷
이 규칙은 기본 생존형 기준. 로그라이크는 "처음부터" 재시작이
핵심 메커닉이므로 별도 설계 (production_spec 2번 항목 참고)
```
