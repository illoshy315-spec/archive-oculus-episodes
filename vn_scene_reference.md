# VN Scene Reference — 극작 기법 레퍼런스
*역전재판 / 단간론파 / 우미네코 / Clannad / Steins;Gate 분석 기반*
*Archive Oculus 전 에피소드 공통 적용*

---

## 이 문서를 쓰는 이유

에이전트가 에피소드 불문하고 반복하는 실수 3가지:
1. **빌런이 자기 계획을 직접 말한다** — 긴장감 소멸
2. **NPC들이 플레이어를 향해 독백만 한다** — 서로 충돌하지 않는다
3. **복선을 "심는" 게 아니라 그냥 "노출"한다** — 나중에 "아!" 하는 순간이 없다

> 이 문서의 기법은 시대/소재/스테이지 구성과 무관하게 작동한다.
> EP별 적용 예시는 `▶ EP 적용 예시` 블록에만 있다. 기법 자체는 범용.

## 파트 1 — 역전재판 (Ace Attorney): 정보를 숨기는 방법

### 1-1. 빌런이 진실을 숨기는 구조

역전재판 원작자 타쿠미의 원칙:
> "플레이어는 작가가 진실을 직접 숨기려 한다는 걸 눈치챈다.
> 그래서 역전재판에서는 *검사(적대자)*가 진실을 숨기는 역할을 맡는다.
> 그게 그 캐릭터의 입장에서 자연스럽기 때문에."

**적용 원칙:**
- 빌런은 자기 계획을 설명하지 않는다. **행동과 질문**으로만 의도를 드러낸다.
- 빌런은 항상 "자신의 목적을 위해 자연스럽게 행동하는 사람"이다.
- 플레이어가 빌런의 의도를 **스스로 계산**하게 만드는 것이 목표.

```
❌ 금지 패턴 — 빌런이 계획을 설명:
  "나는 [플레이어의 자원]을 빼앗을 것이다. [동기]가 있기 때문이다."
  → 긴장감 0. 플레이어가 이미 다 안다.

✅ 올바른 패턴 — 행동만, 말 없음:
  빌런이 [플레이어의 자원과 관련된 물건/장소]에 손을 댄다.
  부하에게 질문한다. "[자원]의 주인이 누구지?"
  대답을 듣는다. 대사 없음. 손을 떼지 않는다.
  → 플레이어가 스스로 연결한다.
```

```
코윈이 지도를 펼친다. 손가락이 강변 쪽을 짚는다.
"강변 쪽 땅 주인이 누구지?"
부하: "엘리자베스... 라고 되어 있습니다."
코윈이 손가락을 떼지 않는다.

→ "코윈이 토지를 노리고 있구나"를 플레이어가 계산한다.
```

### 1-2. 소형 미스터리 루프 (Smaller Mystery Loop)

타쿠미의 핵심 구조:
```
소형 의문 제기 → 소형 해소 → 새로운 전개 → 다음 소형 의문
```

**적용 원칙:**
- 각 씬은 **의문 1개를 열고 1개를 닫는다.**
- 선택지 직전에 미해결 의문이 반드시 1개 이상 떠있어야 한다.
- 의문이 없는 상태에서 선택지가 등장하면 압박감이 없다.

```
씬 설계 템플릿:
  의문 열기:   [NPC의 이상한 행동 또는 모순된 정보]
  씬 진행:     [상황 전개]
  의문 1 닫기: [첫 의문의 표면적 해소]
  의문 2 열기: [그런데 왜 [NPC]는 [행동]을 했지?]
  선택지:      의문 2가 미해결인 상태에서 등장
```

---

### 1-3. 소품 삽입 — 대사 없이 감정을 전달

역전재판이 법정 씬 긴장을 유지하는 방법: **대사와 대사 사이에 NPC 행동 묘사를 끼운다.**

**규칙: 행동은 반드시 대사 내용과 모순되거나, 대사보다 더 많은 정보를 담아야 의미가 있다.**

```
구조:
  NPC 대사 → [행동 묘사 1줄] → 다음 대사

예시 (에피소드 무관):
  "물론이죠, 저는 협력할 의향이 있습니다."
  [NPC가 말하면서 손을 뒤로 감춘다.]
  → 대사는 우호적. 행동이 거짓말을 암시.
  말 따로, 몸 따로.
```

## 파트 2 — 단간론파 (Danganronpa): NPC를 충돌시키는 방법

### 2-1. 플레이어 샌드위치 (Player Sandwich)

단간론파 재판 씬의 핵심: **두 NPC가 충돌할 때 플레이어는 그 사이에 끼인다.**
이 구조가 압박을 만든다. NPC들이 각자 플레이어한테만 말하면 압박이 없다.

```
❌ 독백 구조 (압박 없음):
  NPC A: "[플레이어]를 고발합니다."
  NPC B: "[플레이어]님 편입니다."
  → NPC들이 따로따로 플레이어에게 발표한다.

✅ 충돌 구조 (압박 있음):
  NPC B가 플레이어 쪽으로 다가선다.
  NPC A가 NPC B를 가로막는다. "[B에게] 증언할 용기가 있소?"
  NPC B가 잠깐 멈춘다.
  플레이어는 NPC B의 눈이 흔들리는 걸 본다.
  → NPC끼리 충돌. 플레이어는 그 틈에서 정보를 읽는다.
```

---

### 2-2. 시스템을 캐릭터로 경험시키기 (Kodaka 원칙)

코다카의 핵심:
> "플레이어는 스토리와 게임 시스템을 *캐릭터*를 통해 경험한다.
> 기억에 남는 게임은 기억에 남는 캐릭터에서 온다."

**스탯/상태 변화를 수치로 발표하지 말고 NPC 행동 변화로 보여준다.**

```
스탯 → NPC 반응 변환 원칙:

[스탯명] 경고 단계:
  → 이 에피소드의 핵심 NPC 중 하나가 플레이어를 미묘하게 회피한다

[스탯명] 위험 단계:
  → 복수의 NPC가 플레이어 주변에서 행동을 바꾼다. 명시적 거절.

[스탯명] 붕괴:
  → NPC 전원이 등을 돌린다. 선택지 구조 자체가 바뀐다.
```

```
신앙(30↓): 예배 자리에 플레이어 옆 좌석이 비어있다
평판(60↓): 이웃이 길에서 눈을 마주치지 않는다

```

### 2-3. 레드 헤링 (Red Herring) 배치 규칙

단간론파는 매 챕터마다 최소 1개의 레드 헤링을 설계한다.
레드 헤링은 단순한 함정이 아니라 **그럴듯한 오답**이어야 한다.

```
레드 헤링 설계 원칙:
  1. 오답이 정답보다 직관적으로 더 맞아 보여야 한다
  2. 오답을 선택했을 때 "왜 틀렸는지"가 나중에 납득 가능해야 한다
  3. 에피소드당 최소 1개, 최대 2개 (과하면 플레이어가 아무것도 안 믿음)

설계 템플릿:
  상황: [NPC의 행동]
  레드 헤링: 플레이어가 [오해하는 방향]으로 읽는다
  실제:      [NPC의 진짜 이유]
  반전 순간: [플레이어가 잘못 읽었다는 걸 발견하는 씬]
```

## 파트 3 — 우미네코 (Umineko): 복선을 심는 방법

### 3-1. "보이지 않는 곳에 놓기" 원칙

우미네코의 복선 핵심 규칙:
> **복선은 당시 맥락에서 완전히 자연스러워야 한다.**
> 인위적으로 튀는 복선은 복선이 아니라 예고편이다.

```
❌ 인위적 복선 (예고편):
  내레이터 또는 오르페아가 특정 물건/정보를 "강조"한다.
  → 플레이어가 즉시 "이게 나중에 중요하겠구나"를 안다.
  → 발견의 쾌감이 없다.

✅ 자연스러운 복선:
  NPC가 대화 중 무심코 언급한다. 장면이 곧바로 넘어간다.
  → 당시엔 배경 대사처럼 들린다.
  → 나중에 회수될 때 "그게 그거였어!" 하는 순간이 생긴다.
```

---

### 3-2. 복선 타이밍 3단계

```
타입 A — 조기 심기 (2스테이지 이상 앞):
  형태: NPC 대사 1줄, 배경 묘사, 소품
  목적: "아!" 하는 순간 만들기
  규칙: 당시에 중요해 보이면 안 된다

타입 B — 직전 심기 (바로 앞 씬):
  형태: 명확하지만 의미를 알 수 없는 행동
  목적: 선택지 직전 긴장 고조
  규칙: "이게 뭔가 연결되겠다"는 느낌은 줘도 됨

타입 C — 역방향 복선 (엔딩 이후):
  형태: 오르페아가 에필로그에서 초반 씬을 언급
  목적: "처음부터 알 수 있었는데" 후회감
  규칙: 리플레이 가치를 만드는 장치
```

---

### 3-3. 정보 공개 타이밍 설계

씬 집필 전 각 정보에 대해 결정해야 하는 것:

```
정보 공개 타이밍 결정표 (에피소드별로 채울 것):

| 정보              | 공개 타이밍    | 공개 방식        | 숨기는 이유           |
|------------------|--------------|----------------|----------------------|
| 빌런의 진짜 동기   | [스테이지 X]  | [행동/대사/소품] | 일찍 알면 [부작용]    |
| 페이크 조력자의 배신 | [스테이지 X]  | [미묘한 신호]    | 완전 서프라이즈는 불공정 |
| 진짜 조력자의 정체 | [스테이지 X]  | [즉시 공개 가능] | 알아도 믿기 어려운 상황 |
| 핵심 아이템의 용도 | [최종관문]    | [상황 속 발견]   | 아이템과 용도를 분리   |
```

## 파트 4 — 범용 기법 통합

### 4-1. 압박 쌓기 3단계 공식

모든 스테이지에 적용 가능한 긴장 축적 구조:

```
1단계 — 환경 압박 (배경/공간):
  NPC 대사 없이 공간 자체가 위협적임
  방법: 방청객/군중이 플레이어를 향한다 / 출구가 막혀있다 / 누군가 이미 자리를 비웠다

2단계 — 사회적 압박 (NPC 충돌):
  두 NPC가 대립하고 플레이어는 샌드위치 (파트 2-1 참조)
  방법: 빌런이 조력자를 압박 / 조력자가 플레이어 앞에서 무너진다

3단계 — 내적 압박 (스탯/상태):
  스탯 수치 발표 없이 행동으로 보여줌 (파트 2-2 참조)
  방법: 플레이어가 말하려 할 때마다 빌런이 가로막는다
```

---

### 4-2. Show Don't Tell — 전환 원칙

**직접 말하는 것 → 행동으로 보여주는 것으로 전환:**

| 금지 (직접 말함) | 허용 (행동으로 보임) |
|----------------|-------------------|
| "저는 당신을 배신할 겁니다" | NPC가 플레이어 뒤에서 빌런 측과 눈을 맞춘다 |
| "이 물건이 중요합니다" | NPC가 물건을 건네며 아무 말도 하지 않는다 |
| "당신의 상황이 위험합니다" | 주변 인물들이 하나씩 자리를 피한다 |
| "저 인물은 [부정적 특성]입니다" | 그 인물의 손이 [욕망의 대상]에서 떠나지 않는다 |
| "긴장됩니다" | 플레이어가 [물건/손]을 쥐는 묘사 |

---

### 4-3. NPC 버릇 — 에피소드당 2회 원칙

```
규칙:
  버릇은 에피소드당 NPC마다 최대 2회
  1회차: 소개용 (플레이어가 기억하게)
  2회차: 결정적 순간 (감정 폭발 직전 / 억압 절정)
  그 사이: 버릇 없음 → 억제하는 것처럼 보임

효과:
  1회차를 기억하는 플레이어에게 2회차가 훨씬 무겁게 느껴진다.
  버릇이 매번 나오면 배경 소음이 된다.
```

---

### 4-4. 선택지 직전 5단계 공식

역전재판 + 단간론파 공통 패턴. 선택지 등장 전에 반드시 거친다.

```
1. 공간 확립 (1~2줄):
   어디에 누가 있는지. 물리적 배치.

2. 균형 붕괴 (1줄):
   기대하던 것과 다른 일이 벌어진다.
   예: 판결 직전에 예상 밖의 인물이 개입한다.

3. NPC 반응 (각자 1줄씩):
   서로 다른 NPC가 서로 다른 방향으로 반응 → 충돌 구조.

4. 압박 고조 (1줄):
   플레이어에게 직접 가해지는 압박. 시간 압박 or 시선 압박.

5. 선택지 등장.
```

## 파트 5 — 빌런 엿듣기 씬 범용 재작성 가이드

**에피소드 불문하고 적용 가능한 간접 노출 패턴:**

```
❌ 금지 패턴:
  빌런이 부하와 대화하며 자신의 계획을 설명한다.
  → 플레이어가 빌런의 의도를 이미 안다. 이후 긴장감 없음.

✅ 올바른 패턴:

  [공간 확립]
  빌런이 혼자 또는 부하와 있다.
  플레이어는 이 장면을 엿본다.

  [행동만, 대사 최소화]
  빌런이 [욕망의 대상과 연결된 물건/장소]를 꺼내거나 가리킨다.
  부하에게 질문만 한다. "[대상]의 주인이 누구지?"
  대답을 듣는다.
  빌런이 [물건/장소]에서 손을 떼지 않는다. 대사 없음.

  [빌런 퇴장]
  빌런이 자리를 뜬다.
  플레이어는 연결고리를 스스로 계산한다.

  [오르페아]
  역사적 사실을 비틀어 간접 확인.
  "기록에 따르면, [빌런]은 이 시기에 [욕망의 대상]을 상당히 취득했습니다.
   어떤 방법으로요? 글쎄요. 기록엔 그 부분이 빠져 있군요."
```

## 파트 6 — Clannad / Steins;Gate: NPC를 기능이 아닌 사람으로 만들기

### 6-1. 3레이어 캐릭터 설계 (Clannad 방식)

Clannad의 핵심 원칙: **첫인상이 틀리게 설계하고, 플레이어가 직접 틀렸다는 걸 발견하게 둔다.**

```
레이어 1 (첫인상): 플레이어가 처음 보는 모습
  → 역할 고정형 인상 (강해 보임 / 약해 보임 / 친절해 보임)

레이어 2 (역할):   이야기 속에서 수행하는 기능
  → 빌런 / 조력자 / 배신자 / 중립자

레이어 3 (진짜):   그 행동의 실제 동기
  → 이게 없으면 NPC는 "기능"이지 "사람"이 아니다

NPC 카드 작성 시 레이어 3 필수 확인:
  "이 NPC는 왜 지금 이 행동을 하는가?"
  역할(레이어 2) 때문이 아닌, 인물 동기(레이어 3)로 대답해야 한다.
```

**레이어 3 대사 패턴 — 짧게, 설명 없이:**
```
조력자가 도움을 주다가 잠깐 멈춘다.
"...나도 처음엔 [플레이어와 같은 선택]을 했어요."
그뿐이다. 더 설명하지 않는다.

→ 플레이어가 스스로 계산한다.
  그래서 아는 것이다. 그래서 돕는 것이다.
  말하지 않았지만 다 전달된다.
```

<summary>▶ EP.01 세일럼 레이어 3 예시</summary>

```
NPC    | 레이어 1      | 레이어 2  | 레이어 3
-------|--------------|---------|------------------------
세라   | 차갑고 무서움  | 조력자   | 같은 자리를 먼저 지나온 생존자
메리   | 따뜻한 이웃   | 배신자   | 생존을 위해 누구든 팔 수 있는 공포
코윈   | 냉철한 법 집행 | 빌런     | 토지에 집착하는 탐욕
패리스 | 경건한 지도자  | 중립     | 코윈과의 복잡한 관계 속 생존자
```
</details>

### 6-2. 페르소나 vs 본심 구조 (Steins;Gate 방식)

Steins;Gate 오카베의 설계:
- **겉**: 과대망상적 매드사이언티스트 페르소나
- **속**: 친구를 잃는 것이 세상에서 제일 무서운 평범한 대학생

이 두 층이 **충돌하는 순간**에 캐릭터가 살아있다고 느껴진다.

**에피소드별 NPC 설계 템플릿:**

```
NPC 이름: [         ]
페르소나 (보이는 것): [          ]
본심 (숨겨진 것):     [          ]
충돌 순간:           [어느 씬에서 페르소나가 무너지는가]
```

### 6-3. 같은 상황, 다른 반응 — 살아있는 세계 만들기

Steins;Gate Phone Trigger 원리:
> 같은 사건인데 캐릭터마다 다르게 받아들인다.
> 이 차이가 각 NPC를 "다른 사람"으로 만든다.

**적용 원칙:**
```
하나의 사건/상황에 대해 씬에 등장하는 NPC 전원의 반응을 설계한다.
반응이 동일하거나 비슷하면 그중 하나는 불필요한 NPC다.

반응 설계 템플릿:
  사건: [플레이어가 한 행동 또는 일어난 일]
  NPC A 반응: [행동 1줄] → 이 반응이 암시하는 것: [  ]
  NPC B 반응: [행동 1줄] → 이 반응이 암시하는 것: [  ]
  NPC C 반응: [행동 1줄] → 이 반응이 암시하는 것: [  ]
```

### 6-4. 대사 밀도 3단계

Clannad의 핵심:
> 가장 중요한 감정은 대사로 말하지 않는다.
> 침묵이 관계를 보여준다.

```
밀도 높음 (대사 多):
  언제: 정보 전달 / 스테이지 전환 / 선택지 제시
  역할: 오르페아 멘트 집중 구간

밀도 중간 (대사 + 침묵 교차):
  언제: NPC 첫 만남 / 신뢰-의심 형성
  방법: 대사 1줄 → 행동 묘사 1줄 교차

밀도 낮음 (침묵 優先):
  언제: 감정의 절정 — 배신 확인 / 아이템 회수 / 최종관문 직전
  방법: 핵심 감정을 대사로 말하지 않는다
```

---

### 6-5. NPC 대사 방식 — 캐릭터별 언어 고유성

원칙:
> 다른 NPC가 같은 대사를 해도 어색하지 않으면 실패.

**NPC 카드에 추가할 항목:**
```
말하는 방식:    [문장 구조 / 자주 쓰는 언어 패턴]
절대 하지 않는 것: [이 캐릭터가 절대 쓰지 않는 표현]
이유:          [배경/동기에서 오는 언어 습관]
```

<details>
<summary>▶ EP.01 세일럼 NPC 대사 방식 예시</summary>

```
코윈:  질문 형태로만 압박. 자기 의도 직접 발화 없음. (법관 = 질문으로 통제)
세라:  짧은 문장. 주어 생략. 감정 단어 없음. (생존자는 감정을 아꼈다)
메리:  상냥하지만 "우리 마을" "이웃" 반복. (공동체를 방패로 쓰는 사람)
패리스: 신앙 언어가 매 문장에. 개인적 욕망 표현 없음. (언어 자체가 방어막)
```
</details>

### 6-6. 플레이어-NPC 상호작용 5유형

모든 에피소드에서 씬 설계 시 참조:

```
유형 1 — 테스트 (NPC가 플레이어를 시험):
  NPC가 먼저 요청/질문 → 플레이어 선택이 NPC 관계에 영향
  설계: "이 NPC가 플레이어에게 무엇을 요구하는가?"

유형 2 — 폭로 (NPC가 자신을 의도치 않게 드러냄):
  긴장 상황에서 NPC가 통제를 잃고 진짜 모습이 나옴
  설계: "어떤 상황에서 이 NPC의 페르소나가 무너지는가?"

유형 3 — 도움 요청 (NPC가 플레이어에게 의존):
  NPC에게 약점이 있어야 플레이어가 "관계"를 느낌
  설계: "이 NPC가 플레이어에게 원하는 것은 무엇인가?"

유형 4 — 갈등 (NPC와 플레이어의 방법이 충돌):
  같은 목표인데 수단이 달라서 충돌
  설계: "이 NPC의 방식이 플레이어의 도덕 감각과 어떻게 충돌하는가?"

유형 5 — 반전 (NPC에 대한 판단이 뒤집힘):
  플레이어가 NPC를 잘못 읽었다는 걸 발견
  설계: "어느 씬에서 플레이어의 판단이 틀렸다는 게 드러나는가?"
```

```
[정보 설계]
```

## 파트 8 — Fate/stay night: 정보 전달을 친교로 만드는 방법

*Type-Moon / Kinoko Nasu, 2004*

### 8-1. "부엌 씬" 원칙 — 세계관 설명을 일상으로 포장

Fate/stay night의 주인공 시로는 요리를 잘한다. 그리고 게임의 상당 분량이 부엌에서 벌어진다. 이걸 비판하는 리뷰어들조차 인정하는 것: **중요한 정보가 밥 먹으면서 나온다.**

왜 이 방식이 작동하는가:
- 일상 행동(요리, 식사, 청소) 중에 나오는 대화는 "설명"이 아니라 "대화"처럼 느껴진다
- NPC가 먼저 먹고, 그다음 말한다 — 물리적 상태가 먼저, 정보는 그다음
- 정보를 받는 플레이어(시로)가 모르는 상태이므로 질문이 자연스럽다

**핵심 구조: 무지한 플레이어 = 자연스러운 질문자**

```
설명 씬 금지 패턴:
  NPC가 플레이어에게 직접 세계관을 설명한다.
  "이 시대에는 [제도/법/관습]이 있었습니다. 따라서..."
  → 교과서. 플레이어가 졸린다.

자연스러운 정보 전달 패턴:
  플레이어가 [일상 행동] 중에 이상한 것을 발견한다.
  "왜 [X]인 거죠?"
  NPC가 [행동하면서] 대답한다. 짧게.
  플레이어가 다음 질문을 한다.
  → 대화가 이어지면서 세계관이 조각조각 들어온다.
```

**Archive Oculus 적용 원칙:**
- 오르페아가 역사 배경을 설명하는 대신 → 플레이어가 이상한 것을 발견하고 오르페아에게 묻는다
- 정보는 플레이어의 행동/선택 직후에 나온다 — "왜 그게 틀렸는지"를 그 순간 알게 된다
- 한 번에 전달하지 않는다. 조각을 흩뿌린다.

---

### 8-2. 공원 벤치 씬 — 친교가 쌓이는 구조

Fate에서 시로와 세이버가 공원에서 쉬는 씬. 전투 중간에 갑자기 삽입된다.
겉으로는 아무것도 일어나지 않는다. 그런데 이 씬이 끝나면 플레이어는 세이버를 더 좋아한다.

왜인가:

```
공원 씬 구조 분석:

1. 긴장 해제: 직전까지 긴박했던 상황이 갑자기 느슨해진다
2. 취약성 노출: NPC가 강한 척 하다가 작은 것에 흔들린다
   (세이버: 공원의 아이들을 본다. 잠깐 멈춘다.)
3. 플레이어가 먼저 말을 건다: 강요하지 않고 옆에 있는다
4. NPC가 조금 열린다: 많이 말하지 않는다. 핵심 한마디.
5. 둘 다 말하지 않는 것이 더 많다.

→ "이 NPC와 함께 있고 싶다"는 감각이 생긴다.
  이게 친교다. 정보 전달이 아니다.
```

**Archive Oculus 적용 — 긴장 해제 씬 삽입 원칙:**

```
규칙: 스테이지와 스테이지 사이에 짧은 "숨 돌리기 씬"을 넣는다.
  - 분량: 3~5줄. 길면 안 된다.
  - 내용: NPC가 뭔가 예상 밖의 행동을 한다 (평소와 다른 모습)
  - 목적: 플레이어가 NPC를 "사람"으로 인식하는 순간

금지: 이 씬에서 중요한 정보를 전달하려 하면 안 된다.
      "숨 돌리기"는 이야기와 무관해야 의미가 있다.

예시 템플릿:
  [장소: 이동 중 / 대기 중 / 전환 공간]
  [NPC가 예상 밖의 작은 행동을 한다]
  [플레이어가 반응한다]
  [NPC가 짧게 답한다 — 혹은 대답하지 않는다]
  [장면 전환]
```

---

### 8-3. 노출형 설명 vs 발견형 설명

Fate/stay night에서 배우는 반면교사:
**설명이 너무 직접적이면 플레이어를 수동적으로 만든다.**

Fate의 좋은 부분과 나쁜 부분을 같이 보면 원칙이 나온다:

```
❌ Fate의 실패 패턴 (노출형):
  NPC가 마법 체계를 몇 페이지에 걸쳐 직접 설명한다.
  → "설명"이다. 플레이어는 받아쓴다.

✅ Fate의 성공 패턴 (발견형):
  시로가 아처의 행동에서 이상함을 느낀다.
  시로의 내면에서 질문이 생긴다.
  그 질문이 나중에 씬에서 조각조각 해소된다.
  → 플레이어가 "발견"한다.

발견형 정보 전달 3단계:
  1. 이상함 심기: NPC의 행동/대사에 작은 모순이나 의문을 넣는다
  2. 보류:         당장 해소하지 않는다. 플레이어가 기억하게 둔다
  3. 발견:         나중에 씬에서 자연스럽게 해소된다
```

---

## 파트 9 — 999: Nine Hours, Nine Persons, Nine Doors: 자연스러운 복선 + 대화 중 정보 분산

*Kotaro Uchikoshi / Chunsoft, 2009*

### 9-1. "퍼즐 룸 대화" — 행동 중 정보 전달

999의 핵심 구조: 탈출 퍼즐을 풀면서 등장인물들이 대화한다.

이 구조가 만드는 효과:
> 정보가 "설명"으로 오지 않는다. **함께 무언가를 하는 중에 나온다.**

```
999 정보 전달 패턴:

상황: 등장인물들이 퍼즐을 풀고 있다
  캐릭터 A가 뭔가를 조작하면서 말한다: "이게 왜 이렇게 설계됐지?"
  캐릭터 B가 옆에서 도우면서 대답한다: "배 구조가 [X]라서 그래."
  캐릭터 C가 끼어든다: "그리고 그게 [Y]랑 연결돼."
  → 세계관/배경이 행동 사이에 자연스럽게 삽입된다.
  → 플레이어는 정보를 받는다고 느끼지 않는다.
```

**Archive Oculus 번역:**

```
정보가 나올 타이밍:
  - 플레이어가 선택지를 고민하는 중
  - NPC가 다른 일을 하면서
  - 두 NPC가 의견을 나누는 중 (플레이어가 방청자)

정보가 나오면 안 되는 타이밍:
  - NPC가 플레이어를 정면으로 보고 서있을 때
  - 아무것도 일어나지 않는 공백 시간
  - 오르페아 멘트 직전 "설명 구간"으로 처리할 때
```

---

### 9-2. "사소한 것에 숨긴 복선" (999의 정수)

999의 복선은 정보 전달보다 훨씬 미묘한 곳에 숨겨진다.

유명한 예: 주인공이 에이스라는 캐릭터에게 "클로버와 스네이크가 남매라서 놀랐다"고 말한다. 에이스는 "그렇군요. 남매인데 외모가 다른 경우도 있죠"라고 답한다.

나중에 밝혀지는 것: 에이스는 안면인식장애가 있다. 그래서 두 사람이 다르게 생겼는지 몰랐던 것이다.

**이 복선이 작동하는 이유:**
- 당시에는 완벽히 자연스러운 대화였다
- 나중에 다시 보면 전혀 다른 의미다
- 복선이 "중요해 보이는 곳"에 없다

**Archive Oculus 번역 — 자연스러운 복선 심기 공식:**

```
복선 삽입 위치 4가지 (중요해 보이지 않는 곳):

1. 잡담 중: NPC들이 주제와 무관해 보이는 대화를 나누다가 한마디 지나간다
2. 이동 중: 장소를 옮기는 씬 전환 직전 대사
3. 반응 묘사: NPC가 뭔가에 "이상하게" 반응하지만 설명하지 않는다
4. 플레이어 내면: 오르페아가 아닌 플레이어 시점에서 "이상하다"는 느낌만 남긴다

복선 확인 기준:
  이 복선이 당시 맥락에서 완전히 자연스러운가?
  → YES: 올바른 복선
  → NO:  힌트 박스. 다시 쓸 것
```

### 9-3. 캐릭터 관계를 통한 정보 공개

999가 정보를 전달하는 방법: **직접 설명하지 않고, 캐릭터 사이의 긴장에서 나온다.**

```
정보 공개 3가지 방식 비교:

방식 A (설명형, 금지):
  NPC가 플레이어에게 직접 설명한다.
  "이 배에는 [X]가 있고, [Y]이기 때문에 [Z]입니다."

방식 B (갈등형, 권장):
  두 캐릭터가 [X]에 대해 의견이 다르다.
  그 갈등 과정에서 플레이어가 [X]를 이해하게 된다.

방식 C (발견형, 권장):
  플레이어가 [X]와 관련된 물건/상황을 발견한다.
  NPC가 반응한다. 짧게.
  플레이어가 스스로 [X]를 추론한다.

Archive Oculus 적용:
  방식 B: 역사적 사실을 두 NPC의 관점 차이로 전달
           예) 당시 법을 따르는 NPC vs 현대 감각의 플레이어
  방식 C: 물건/현장을 먼저 보여주고 설명은 나중에
```

## 파트 10 — 추가된 기법 통합: 친교 + 자연스러운 정보 전달 + 미묘한 복선

## 파트 11 — 아이템 설계: 유용한 것, 쓸모없는 것, 함정

*역전재판 타쿠미 강연 + 어드벤처 게임 설계 원칙 기반*

### 핵심 질문 3가지

```
1. 아이템을 어떻게 심어야 나중에 "아!"가 생기는가?
2. 쓸모없는 아이템을 어떻게 써야 탐색 의욕이 살아있는가?
3. 함정 아이템은 어떻게 설계해야 "당했다"가 아니라 "납득"이 오는가?
```

### 11-1. 역전재판의 레드 헤링 아이템 철학

타쿠미의 직접 발언 (GCC '18 강연):
> "조사 단계에서 레드 헤링을 건드렸을 때 단순히 '아무것도 없습니다'라고 뜨면 플레이어의 탐색 의욕이 꺾인다. 레드 헤링을 건드렸을 때도 **보상이 있어야 한다.** 나는 레드 헤링 반응 텍스트도 어드벤처 게임의 핵심 요소라고 생각한다."

이것이 의미하는 것:

```
쓸모없는 아이템 = "보상 없는 탐색"이 아니라
쓸모없는 아이템 = "다른 종류의 보상이 있는 탐색"

보상의 종류:
  A. 캐릭터 반응: 오르페아가 그 아이템에 한마디 던진다 (냉소적으로)
  B. 세계관 조각: 아이템 자체가 시대 배경을 알려주는 작은 디테일
  C. 복선 노출: 당시엔 의미없어 보이지만 나중에 연결된다 (타입 D 복선)
  D. 감정 보상: 재미있거나 기묘하거나 불편한 반응 — 기억에 남는다
```

**Archive Oculus 적용:**
```
아이템/상황에 플레이어가 주목했을 때:
  - "이건 중요한 게 아니에요"로 끝내지 않는다
  - 오르페아가 반드시 1줄 반응한다
  - 그 1줄이 세계관 디테일이거나, 냉소이거나, 미래 복선이어야 한다

예시 템플릿:
  [플레이어가 무관한 물건에 주목]
  오르페아: "[아이템에 대한 냉소/시대 맥락/의미심장한 한마디]"
  → 쓸모없는 탐색도 "보람 있는 탐색"이 된다
```

### 11-2. 아이템 4가지 유형 — 설계 분류표

```
유형 1 — 열쇠 아이템 (Key Item):
  정의: 특정 엔딩/선택지를 활성화하는 조건 아이템
  특징: 획득 시점과 사용 시점이 분리된다
  설계 규칙: 획득 씬에서 용도를 암시하되 직접 말하지 않는다
  위험: 너무 일찍 용도를 알려주면 최종관문에서 긴장감 없음

유형 2 — 정보 아이템 (Info Item):
  정의: 물건이 아닌 "알게 된 사실"
  특징: 인벤토리에 텍스트로만 존재
  설계 규칙: 어디서 얻었는지(획득 씬)가 중요 — 정보의 출처가 신뢰도를 결정
  위험: 정보를 너무 완전하게 주면 플레이어가 계산할 게 없음

유형 3 — 레드 헤링 아이템 (Red Herring):
  정의: 그럴듯해 보이지만 실제로 사용되지 않는 아이템
  특징: 존재 자체가 함정 — 플레이어가 이걸 믿으면 잘못된 선택을 한다
  설계 규칙: 반드시 "보상 있는 탐색"으로 만들 것 (타쿠미 원칙)
  위험: 보상 없이 그냥 쓸모없으면 플레이어가 탐색 자체를 포기

유형 4 — 타이밍 아이템 (Conditional Item):
  정의: 조건이 맞을 때만 작동하는 아이템
  특징: 같은 아이템이 특정 스탯/선택지 조합에서만 유효
  설계 규칙: 조건이 명시적이지 않아야 발견의 쾌감이 생김
  위험: 조건이 너무 숨겨지면 "알 수 없는 게임"이 됨
```

### 11-3. 아이템 획득 씬 설계 — 3가지 패턴

아이템을 "그냥 줍는" 방식은 서사적으로 죽은 씬이다.

```
패턴 A — 갈등 끝에 획득 (가장 강함):
  무언가를 결정하거나 위험을 감수한 결과로 얻는다
  획득 자체가 플레이어의 선택을 반영한다
  → 아이템에 "내 선택의 무게"가 붙는다

  구조:
  [딜레마 상황] → [선택] → [획득] or [소실]
  둘 다 가능한 선택지에서 B를 골라야 획득되는 구조가 이상적

패턴 B — NPC에게서 획득 (관계 반영):
  NPC가 아이템을 줄 때 그 관계의 현재 상태가 보인다
  냉랭하게 주는가 / 따뜻하게 주는가 / 몰래 주는가
  → 아이템에 "이 NPC와의 관계"가 붙는다

  구조:
  [NPC 감정 상태 묘사] → [건네는 방식 묘사] → [획득]
  대사보다 "건네는 행동"이 더 중요

패턴 C — 발견형 획득 (가장 자연스러움):
  탐색하다가 "이게 있었나?" 하고 발견
  처음에 중요해 보이지 않는다
  → 나중에 쓸 때 "그게 그거였어!" 효과

  구조:
  [무관해 보이는 씬] → [우연한 발견] → [오르페아 반응 1줄]
  → 회수 씬에서 비로소 의미 부여
```

### 11-4. 함정 아이템 설계 — "당했다"가 아닌 "납득"을 만드는 방법

함정 아이템의 목표: 플레이어가 "속았다"고 느끼는 게 아니라 **"아, 그래서 그랬구나"** 를 경험하게 하는 것.

```
나쁜 함정 아이템 (분노 유발):
  아이템이 그냥 쓸모없다
  혹은 아이템을 믿은 결과가 "게임 디자이너에게 속은" 느낌
  → 플레이어가 시스템을 불신하게 됨

좋은 함정 아이템 (납득 유발):
  아이템이 틀린 이유가 역사적/논리적으로 납득 가능하다
  플레이어의 현대 상식이 틀린 것이지, 게임이 속인 것이 아니다
  → "내가 모른 것" + "역사적 사실" = 납득

Archive Oculus 전용 함정 아이템 설계 원칙:
  함정 아이템은 반드시 "현대 상식으로 합리적이지만 당시엔 통하지 않는" 것이어야 한다
  아이템 자체가 나쁜 게 아니라 → 사용 맥락이 틀린 것
  배드 엔딩에서 "왜 안 통했는지"가 역사적 사실로 설명돼야 한다
```

**함정 아이템 설계 체크리스트:**

```
□ 이 아이템을 현대인이 "당연히 도움이 된다"고 믿겠는가?
  → YES여야 함. 그래야 함정으로 작동.

□ 이 아이템이 통하지 않는 이유가 역사적으로 명확한가?
  → YES여야 함. 게임이 아니라 역사가 함정을 설계한 것.

□ 배드 엔딩에서 "왜 틀렸는지"를 오르페아가 설명하는가?
  → YES여야 함. "당했다"가 아니라 "배웠다"가 돼야 함.

□ 함정 아이템과 진짜 아이템이 표면상 비슷하게 보이는가?
  → YES여야 함. 구분이 너무 쉬우면 함정이 아님.
```

### 11-5. 아이템 회수 씬 — 사용 순간이 획득보다 중요하다

아이템의 감정적 무게는 **획득 씬**이 아니라 **사용 씬**에서 완성된다.

```
회수 씬 설계 3원칙:

원칙 1 — 타이밍의 극적 지점:
  아이템을 꺼내는 순간이 씬의 클라이맥스여야 한다
  선택지 안에 아이템 사용 조건이 있을 때:
  그 조건이 만족되는 순간을 씬으로 연출한다

원칙 2 — 획득 씬 연결:
  회수 씬에서 획득 씬을 짧게 환기시킨다
  플레이어가 "그때 그거를 선택했기 때문에"를 느끼게
  방법: 동일한 묘사어, NPC의 같은 버릇, 같은 장소 디테일 재사용

원칙 3 — 결과의 이중성:
  아이템을 썼는데 완벽한 해결이 되면 안 된다
  반드시 비용이 따른다 — 무언가를 얻으면 무언가를 잃는다
  → 아이템 사용이 "쉬운 정답"이 아니라 "도덕적 무게"가 됨
```

```
  세라의 목소리가 플래시백으로 겹친다

회수 원칙 2 적용:
```

### 11-6. 아이템 설계 통합 체크리스트

```
[회수 씬]

□ 획득 씬이 환기되는 요소가 있는가?

```

### 12-1. 오프닝 시그니처 — 고정 구조

매 에피소드 동일한 포맷. 내용만 변주.

```
[시네마틱]
VR 헤드셋을 착용하는 손. 클로즈업.
헤드셋이 눈을 덮는다.

[로딩 화면 — 시스템 텍스트]
오르페아 (차갑고 기계적인 톤):
"Archive Oculus. 역사 생존 시뮬레이션."
"인스턴스 로드 중."
"[에피소드 제목 / 연도 / 장소]"
"경고: 이 시뮬레이션에서 당신의 선택은 기록됩니다."

[접속 선언]
오르페아:
"[에피소드명]에 접속했습니다."

[방향 A 또는 방향 B로 분기]
```

**고정 요소:** 헤드셋 시네마틱 / 로딩 텍스트 / 접속 선언
**변주 요소:** 에피소드 제목, 연도, 장소, 경고 문구 (에피소드마다 다르게)

경고 문구는 에피소드 핵심 테마를 암시할 수 있다:
```
EP.01 세일럼: "경고: 이 시뮬레이션에서 침묵은 유죄의 증거입니다."
EP.XX 빅토리아: "경고: 이 시뮬레이션에서 신분은 생존 확률과 비례합니다."
→ 경고 문구가 이미 첫 번째 복선이 된다.
```

### 12-2. 두 방향 선택 기준

접속 선언 직후, 에피소드 성격에 따라 방향을 고른다.

```
방향 A — 현장 직행 (In Media Res)
방향 B — 낭만 파괴 (Expectation Reversal)
```

**방향 A를 고를 때:**

| 조건 | 이유 |
|------|------|
| 역사적 사건이 이미 유명한 경우 | 설명이 불필요. 바로 들어가는 게 더 강하다 |
| 오프닝 상황 자체가 충격적인 경우 | 현장이 이미 훅이다 |
| 빠른 템포가 필요한 에피소드 | 도입을 길게 끌면 긴장이 풀린다 |
| 플레이어가 이미 상황에 던져진 설정 | 설명할 시간 자체가 없는 상황 |

**방향 B를 고를 때:**

| 조건 | 이유 |
|------|------|
| 역사적 사건이 생소하거나 로망이 있는 경우 | 기대를 부풀려야 파괴의 낙차가 생긴다 |
| 인지 부조화가 핵심 훅인 경우 | 기대와 현실의 격차가 클수록 임팩트 |
| 시청자에게 감정 투자 시간이 필요한 경우 | 세계를 먼저 사랑하게 해야 잃는 게 아프다 |
| 장르 클리셰를 역이용하는 경우 | "이런 거 알잖아요" → "아니요, 틀렸어요" |

---

### 12-3. 방향 A — 현장 직행 포맷

```
[접속 선언 직후]
오르페아: "[에피소드명]에 접속했습니다."

[즉시 현장]
설명 없이 플레이어는 이미 상황 안에 있다.
첫 문장부터 감각 정보 — 소리, 냄새, 온도.
"어디에 있는지"보다 "지금 무슨 일이 벌어지는지"가 먼저.

[오르페아가 상황을 소개하되 설명하지 않는다]
오르페아: "[장소]. [연도]. [플레이어의 신분 — 1문장]"
"생존 확률이 계산되고 있습니다."

[첫 번째 위협 또는 긴장 요소가 즉시 등장]
```

**작성 규칙:**
```
- 첫 문장에 시각 외 감각 1개 이상
- "당신은 ~에 있습니다" 형식의 설명 금지
- 플레이어가 먼저 느끼고, 나중에 파악한다
- 오르페아의 첫 멘트는 상황 파악을 돕되 긴장을 낮추지 않는다
```

### 12-4. 방향 B — 낭만 파괴 포맷

```
[접속 선언 직후]
오르페아: "[에피소드명]에 접속했습니다."

[낭만 제시 — 15~20초 분량]
플레이어가 상상할 법한 아름다운/낭만적/흥미로운 이미지 먼저.
시청자가 "맞아, 그 시대 그거잖아"라고 끄덕이는 이미지.
오르페아의 어조가 약간 다르다 — 마치 그 낭만을 인정하는 것처럼.

[파괴 — 즉각적, 감각적]
"그런데."
또는 오르페아가 말 없이 현실을 들이민다.
낭만과 180도 반대되는 감각 정보 (냄새, 소리, 촉감).

[현실 선언]
오르페아가 낭만이 틀렸음을 냉소적으로 확정.
플레이어 신분 부여.
생존 확률 제시.
```

**작성 규칙:**
```
- 낭만 구간은 반드시 시청자의 기존 상상과 일치해야 한다
  (틀린 낭만이면 파괴가 작동하지 않는다)
- 파괴는 감각으로 — 설명이 아니라 냄새/소리/온도로 들이민다
- 낭만 → 파괴의 전환점에 오르페아 멘트가 있어도 되고 없어도 된다
  (침묵으로 파괴하는 것이 더 강할 때도 있다)
- 낭만 구간이 너무 길면 파괴의 낙차가 줄어든다. 15~20초가 최대.
```

### 12-5. 경고 문구 — 에피소드별 첫 번째 복선

경고 문구는 단순한 시스템 메시지가 아니다.
에피소드의 핵심 함정 또는 테마를 미리 암시하는 **첫 번째 복선**이다.

```
설계 원칙:
  - 당시에는 시스템 경고처럼 들린다
  - 에피소드가 끝나고 나면 "그게 그 의미였구나"가 된다
  - 직접적 힌트가 아니라 분위기/규칙 암시

경고 문구 작성 기준:
  1. 이 에피소드에서 현대 상식이 통하지 않는 핵심 규칙이 무엇인가?
  2. 그걸 "시스템 경고" 어투로 1문장에 담는다
  3. 이해했다면 생존 확률이 올라가는 문장이어야 한다

에피소드별 예시 템플릿:
  "경고: 이 시뮬레이션에서 [현대 상식의 핵심 오류]는 [결과]로 이어집니다."
```

## 파트 13 — 페이스 컨트롤: 에피소드 전체 긴장 곡선 설계

### 13-1. 핵심 원칙: 페이스는 속도가 아니라 압력이다

> "스릴러가 좋은 건 빠르기 때문이 아니다. 긴장되기 때문이다.
> 문학 소설이 지루한 건 느리기 때문이 아니다. 느슨하기 때문이다."

**페이스 컨트롤의 목표:**
플레이어가 숨막히는 순간과 숨 쉬는 순간을 교대로 경험하게 한다.
계속 조이기만 하면 감각이 마비된다.
계속 풀기만 하면 긴장이 없다.

---

### 13-2. Archive Oculus EP 긴장 곡선 표준 모델

```
[오프닝 훅]     ██████ 높음    → 즉각 투입, 궁금증 발생
[매치업]        █████  중상    → 상황 파악, 위협 도입
[스테이지 1]    ████   중간    → 첫 선택지, 적당한 압박
[숨 돌리기 A]   ██     낮음    → 3~5줄, NPC 다른 면
[스테이지 2]    █████  중상    → 긴장 상승
[스테이지 3]    ██████ 높음    → 절정 직전, 최고 압박
[숨 돌리기 B]   █      최저    → 선택지 직전 의도적 정지
[최종관문]      ███████ 최고   → 클라이맥스
[엔딩]          ████   중간    → 감정 착지, 여운
```

**핵심 규칙:**
```
① 스테이지마다 이전보다 높아야 한다 (단조 상승 금지 — 숨 돌리기로 낙차 만들기)
② 최종관문 직전에 의도적 정지가 있어야 한다 (숨 돌리기 B)
③ 클라이맥스 직전 정지가 클라이맥스를 더 강하게 만든다
④ 엔딩은 클라이맥스보다 낮아야 한다 — 감정이 내려와야 여운이 생긴다
```

### 13-3. 긴장 올리는 방법 vs 내리는 방법

```
긴장을 올리는 방법:
  - 선택지 직전 5단계 구조 적용 (파트 4-4)
  - 스탯 위험 단계 진입 NPC 반응 변화
  - 새 위협 등장 또는 기존 위협 확대
  - 시간 압박 또는 시선 압박 추가
  - 대사 밀도 높임 — 짧은 문장, 빠른 교환
  - 감각 묘사 강도 상승 (냄새/온도/소리 더 구체적으로)

긴장을 내리는 방법:
  - 숨 돌리기 씬 삽입 (파트 8-2, 3~5줄)
  - 대사 밀도 낮춤 — 침묵 우선
  - 장소 전환 (실내→야외, 밀폐→개방)
  - NPC의 예상 밖 인간적 모습 1개
  - 오르페아가 잠깐 냉소를 거두는 순간
```

---

### 13-4. 단간론파가 가르쳐주는 것: "폭풍 전 고요"

단간론파는 가장 충격적인 장면 **직전**에 의도적으로 분위기를 낮춘다.
캐릭터들이 웃고, 가벼운 대화를 나누고, 희망적인 분위기가 된다.
그리고 바로 그 다음 씬에서 살인이 일어난다.

이 구조가 주는 효과:
> "방심하게 만든 다음 치는" 충격이 "계속 긴장시키다 치는" 충격보다 훨씬 강하다.

**Archive Oculus 적용:**
```
최종관문 직전 숨 돌리기 B는 단순한 휴식이 아니다.
플레이어를 방심시키는 의도적 장치다.

숨 돌리기 B 작성 원칙:
  - 분위기가 잠깐 풀린다 (NPC가 의외로 인간적이다)
  - 플레이어가 "어, 생각보다 괜찮을 수도?"라고 느낀다
  - 그 다음 최종관문이 더 강하게 작동한다

이것이 작동하려면 숨 돌리기 B가 진짜처럼 느껴져야 한다.
작위적이면 안 된다. 플레이어가 함정인 줄 모르게.
```

---

### 13-5. 미스터리 vs 서스펜스 — 긴장의 두 가지 종류

Alfred Hitchcock의 구분:

```
미스터리 (Mystery):
  플레이어가 모른다 → 궁금해서 따라간다
  "폭탄이 터진다. 플레이어도 몰랐다."
  효과: 짧은 충격

서스펜스 (Suspense):
  플레이어는 안다. 캐릭터는 모른다 → 발을 동동 구르며 따라간다
  "폭탄이 테이블 아래 있다. 플레이어는 안다. 캐릭터는 모른다."
  효과: 지속적 긴장
```

**Archive Oculus 적용:**
```
Archive Oculus는 두 가지를 모두 쓴다:

미스터리 구간: 오르페아가 아직 알려주지 않은 것
  → 빌런의 진짜 동기 / 숨겨진 복선 / 선택지의 함정

서스펜스 구간: 오르페아가 이미 알려준 것 (생존 확률, 역사적 결과)
  → "이미 알고 있다. 그런데 캐릭터는 모른다."
  → "생존 확률 23%"를 알고 있는 상태에서 선택지를 본다

오르페아의 냉소가 서스펜스를 만든다:
  오르페아가 "역사적으로 이 선택은 사망률이 높습니다"라고 말한다
  → 플레이어는 안다. 캐릭터는 모른다. 긴장이 생긴다.
```

## 파트 14 — 선택지 텍스트 작성법: 함정을 그럴듯하게 만드는 기술

### 14-1. 선택지의 3가지 목표

```
목표 1 — 매력: 모든 선택지가 고를 만해 보여야 한다
목표 2 — 차별: 각 선택지가 서로 다른 가치관/접근법을 대표해야 한다
목표 3 — 함정: 현대 상식 선택지가 역사적으로 틀려야 한다
```

**Archive Oculus의 특수 조건:**
함정 선택지가 항상 "현대 상식 = 합리적으로 보임"이어야 한다.
현대인이 고를 법한 선택지가 역사적으로 죽음의 길이라는 반전.

---

### 14-2. 함정 선택지 문장 작성 4원칙

```
원칙 1 — 현재 시제, 1인칭 행동:
  "나는 [행동]한다" 형식
  능동적이고 자신감 있어 보여야 한다
  ❌ "어쩌면 결백하다고 말해볼 수도 있다"
  ✅ "결백합니다. 물리적 증거를 요구합니다"

원칙 2 — 현대 상식의 언어:
  함정 선택지는 현대 드라마/영화의 주인공이 할 법한 말이어야 한다
  "증거를 요구한다" "변호사를 요청한다" "권리를 주장한다"
  → 현대인에게 완전히 합리적인 언어

원칙 3 — 도덕적 우위를 가진 것처럼 보이기:
  함정 선택지가 "옳은 것"처럼 느껴져야 한다
  진실을 말하는 것 / 원칙을 지키는 것 / 당당한 것
  → 고르지 않으면 비겁한 것처럼 느껴지는 유혹

원칙 4 — 탈출 선택지는 불편하게:
  정답 선택지가 불편하거나 비겁해 보여야 한다
  "자백한다" "공범을 댄다" "도망친다"
  → 현대 감각으로는 나쁜 선택처럼 보여야 함
```

---

### 14-3. 선택지 유형 4가지

```
유형 A — 역사적 함정 선택지:
  현대 상식으로 정답이지만 당시엔 사형 선고
  작성 방법: 현대 드라마 주인공이 할 말 그대로
  오르페아 평가: "역사적으로 인기 있는 자살 루트입니다"

유형 B — 탈출구 선택지:
  불편하고 비겁해 보이지만 실제로 살아남는 길
  작성 방법: 현대 감각으로 "이거 하면 나쁜 사람 같은데"가 되는 것
  오르페아 평가: "꽤 비열한 방법입니다. 그래서 효과적이죠"

유형 C — 중립 선택지:
  큰 위험도 없고 큰 보상도 없는 안전한 선택
  작성 방법: 아무것도 하지 않거나 최소한만 하는 것
  오르페아 평가: "소극적이군요. 역사는 이런 사람들을 기억하지 않습니다"

유형 D — 아이템 조건부 선택지:
  아이템/정보 보유 시만 활성화
  작성 방법: 잠금 표시(🗝️)와 함께 조건 명시
  오르페아 평가: 보유자에게만 다르게 표시
```

---

### 14-4. 오르페아 평가 문장 작성 원칙

```
금지:
  "정답입니다" "오답입니다" "잘하셨습니다" "현명하군요"

허용:
  역사적 결과를 우회적으로 암시
  플레이어의 감정(자존심/두려움/합리성)을 비틀기
  냉소적이지만 틀린 말이 아닌 것

함정 선택지 평가 공식:
  "[선택 동기를 인정] + [그러나 결과는 다르다] + [오르페아 특유의 비틀기]"

  예시:
  "당당하군요. 역사는 당당한 사람들을 기록했습니다."
  "[잠깐 멈춤]"
  "사망자 목록에."

탈출구 선택지 평가 공식:
  "[선택의 불편함 인정] + [그래서 효과적이라는 반전] + [씁쓸한 한마디]"

  예시:
  "비겁합니다. 역사도 그렇게 기록했을 겁니다."
  "하지만 기록은 살아있는 사람이 씁니다."
```

## 파트 15 — 엔딩 씬 설계: 감정 착지

### 15-1. Archive Oculus 3종 엔딩의 감정 목표

```
사망 엔딩 (Death):
  감정 목표: "납득 가는 비극" — 분노가 아닌 씁쓸함
  핵심: 플레이어가 "내가 틀렸구나"를 느껴야 한다
        "게임이 불공평하다"를 느끼면 실패

일반생존 엔딩 (Normal):
  감정 목표: "찜찜한 안도" — 살았지만 뭔가 잃었다
  핵심: 완전한 승리가 아니다
        도덕적 비용이 있어야 한다 (누군가를 희생시켰다)

퍼펙트 엔딩 (Perfect):
  감정 목표: "차가운 승리" — 기쁘지만 완전히 기쁘지 않다
  핵심: 오르페아가 축하하지 않는다
        역사적 아이러니로 마무리한다
```

---

### 15-2. 각 엔딩 씬 구조

**사망 엔딩:**
```
[사망 순간] → 짧고 감각적으로. 설명하지 않는다.
[오르페아] → 역사적 사실로 이유를 설명. 냉정하게.
             "기록에 따르면, [당시 법/제도]는 [이유]를 인정하지 않았습니다."
[접속 종료 선언] → "시뮬레이션이 종료되었습니다."
                   "다음 접속을 권장합니다." (재도전 유도)
[오르페아 마지막 한마디] → 플레이어의 현대 감각을 비틀기.
                           "흥미롭군요. 현대인도 같은 선택을 합니다."

금지: 긴 설명 / 감정적 위로 / "다시 해보세요" 직접 권유
```

**일반생존 엔딩:**
```
[생존 확인] → 살았다. 단 1~2줄.
[비용 제시] → 무엇을 잃었는가. NPC 반응 또는 역사적 기록으로.
[오르페아] → "살아남았습니다. [그리고 무엇이 따라왔는가]."
             씁쓸한 사실 1개.
[역사 에필로그] → 실제 역사에서 이런 선택을 한 사람들이 어떻게 됐는지.
[오르페아 마지막] → 도덕적 무게를 한마디로.

핵심: 엔딩이 끝날 때 플레이어가 "잘했다"가 아니라 "복잡하다"를 느껴야 한다.
```

**퍼펙트 엔딩:**
```
[승리 확인] → 명쾌하게. 단 짧게.
[오르페아 축하] → 냉소적 축하. 따뜻하지 않다.
                  "축하드려요, 플레이어. 당신은 살아남았습니다."
                  "[잠깐 멈춤]"
                  "이번에는."
[역사 아이러니] → 이 에피소드에서 가장 충격적인 역사적 사실.
                  "참고로 실제 역사에서는..."
[오르페아의 의미심장한 질문] → 시청자 참여 질문으로 연결.
                               "당신이라면 어떤 선택을 했을까요?"
[채널 참여 유도] → 게임 별점 / 댓글 / 구독을 VR 게임 어투로.

핵심: 완전한 해피엔딩이 아니다. 역사의 무게로 마무리한다.
```

---

### 15-3. 오르페아 엔딩 톤 — 엔딩별 차이

```
사망 엔딩:
  톤: 냉정. 감정 없음. 데이터를 읽는 것처럼.
  오르페아는 슬퍼하지 않는다. 예상된 결과처럼 다룬다.
  → 이게 더 무섭다.

일반생존 엔딩:
  톤: 약간의 냉소 + 씁쓸함.
  "살았군요"라고 하지만 그 다음이 더 무겁다.
  오르페아가 평소보다 조금 더 말을 아낀다.
  → 침묵이 도덕적 무게를 만든다.

퍼펙트 엔딩:
  톤: 의외로 가장 냉소적.
  완벽하게 이겼는데 오르페아는 별로 감흥이 없다.
  "예상 가능한 결과였습니다"라는 뉘앙스.
  → 이 냉소가 역사의 무상함을 표현한다.

공통 금지:
  엔딩에서 오르페아가 플레이어를 칭찬하지 않는다.
  "잘하셨습니다" "훌륭합니다" "현명했어요" 절대 금지.
  오르페아는 결과를 기록할 뿐이다.
```

---

### 15-4. 엔딩 씬 체크리스트

```
[공통]
□ 엔딩이 플레이어의 선택의 직접적 결과로 느껴지는가?
□ "게임이 불공평하다"가 아닌 "내가 틀렸다"로 착지하는가?
□ 역사적 사실이 엔딩을 뒷받침하는가?
□ 오르페아가 칭찬하지 않는가?

[사망 엔딩]
□ 사망 순간이 짧고 감각적인가?
□ 이유 설명이 역사적 사실 기반인가?
□ 재도전 유도가 직접적이지 않은가?

[일반생존 엔딩]
□ 비용이 명확히 제시되는가?
□ "찜찜한 안도"로 착지하는가?
□ 도덕적 무게가 마지막 한마디에 있는가?

[퍼펙트 엔딩]
□ 역사 아이러니가 있는가?
□ 의미심장한 질문으로 마무리하는가?
□ 채널 참여 유도가 VR 게임 어투인가?
□ 완전한 해피엔딩이 아닌가?
```
