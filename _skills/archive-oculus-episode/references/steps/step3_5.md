# Step 3.5 — 서사 설계 + 복선 연결 + scenario.json 완성

## 임무
scenario.json 골격에 복선/아이템/인과관계를 추가하고
에피소드 서사 개요를 완성한다.
이 단계 완료 후 scenario.json을 잠근다.

---

## 파트 구조 원칙

**파트 수는 유동적이다.**
오프닝 + 스테이지 N개 + 최종관문 + 엔딩 구조만 고정.
스테이지 수는 에피소드 분량과 포맷에 따라 결정.

```
오프닝
  ↓
스테이지 1
  ↓ (합류)
스테이지 2
  ↓ (합류)
... (N개)
  ↓ (합류)
최종관문 → 누적 스탯에 따라 진짜 분기
  ↓
엔딩 A (퍼펙트) / 엔딩 B (일반생존) / 엔딩 C (사망) / 히든
```

**포맷별 스테이지 수 기준** (`references/production_spec.md` 참고):
- 기본 생존형 (20~23분): 스테이지 3개
- 로그라이크: 스테이지 2개 (짧고 반복)
- 육성 시뮬레이션: 스테이지 4~5개 (날짜 단위)
- 법정물: 스테이지 2개 (심문 + 최종 변론)

각 스테이지는 다이아몬드 구조 (선택 → 스탯 변화만 다름 → 합류).
최종관문에서만 진짜 분기 발생.

---

## 파트 0: 분기 트리 설계

엔딩 조건 명세 (필수 — scenario.json에 포함):
- 퍼펙트 생존: 구체적 스탯 임계값 (예: 전체 스탯 평균 70 이상)
- 일반 생존: 구체적 스탯 임계값
- 사망: 특정 스탯 0 도달 또는 지뢰 N개 이상 적중
- 히든: 특수 트리거 (예: 특정 스테이지에서 아무것도 선택하지 않음)

---

## 파트 1: 인과관계 설계

각 스테이지 간 연결을 명시한다.
"오프닝/스테이지 X에서 심은 [아이템/정보/관계]가 스테이지 Y 또는 최종관문에서 어떻게 회수되는가"

| 심는 파트 | 심는 것 | 회수 파트 | 회수 방식 |
|-----------|---------|-----------|-----------|
| 오프닝 | [아이템/정보명] | 최종관문 | [어떻게 씀] |
| 스테이지1 | [NPC 관계] | 스테이지3 | [어떻게 씀] |

---

## 파트 2: 에피소드 서사 개요

각 파트마다 아래 항목을 채운다.
스테이지 수는 이 에피소드에서 결정된 수에 맞게 늘리거나 줄인다.

```
[오프닝]
핵심 사건: 접속 + 시대 분위기 설정 + 신분 부여
감각 키워드: [청각] [촉각] [후각]
오르페아 포인트: "[냉소적 한마디]"
심는 아이템/복선: ...
선택지 유형: 없음 (세계관 진입만)

[스테이지 1]
핵심 상황: ...
지뢰 (역사적 함정): [팩트ID]
선택지 유형: [역사적함정 / 인물관련 / 스탯돌발 / 아이템활용]
유혹적 선택 → 결과: ...
탈출구 → 결과: ...
획득 아이템/정보: ...
오르페아 포인트: ...

[스테이지 2]
...

[스테이지 N]
...

[최종관문]
심어둔 아이템/정보 전부 회수
누적 스탯 임계값 체크
선택지 유형: 역사적 함정 집중
배드엔딩 진입 조건: ...

[엔딩]
퍼펙트: 역사적 사실 기반 에필로그 + 오르페아 의미심장한 축하
일반생존: ...
사망: 랜턴 꺼짐 → 오르페아 사망 코멘트 → 재도전 화면
히든: 특수 트리거 조건 + 연출
```

## 파트 3: scenario.json 최종 업데이트

파트 1에서 설계한 인과관계를 scenario.json stages[]의
`planted_item`과 `foreshadow` 필드에 채운다.

## 🔒 scenario.json 잠금

사용자 승인 후 scenario.json을 확정 잠금한다.
이후 Step 4 집필 중 scenario.json 내용은 **변경 불가**.
집필 중 변경이 필요하면 사용자에게 보고하고 재승인을 받는다.
