# Step 3.5 — 서사 설계 + 복선 연결 + scenario.json 완성

## 임무
scenario.json 골격에 복선/아이템/인과관계를 추가하고
7파트 서사 개요를 완성한다.
이 단계 완료 후 scenario.json을 잠근다.

---

## 파트 0: 분기 트리 설계 (추가)

다이아몬드 구조 권장 (완전 자유 분기는 Step 4 집필량 폭발).

```
오프닝
  ↓
스테이지1 (선택 A/B/C) → 스탯 변화만 다름
  ↓ (합류)
스테이지2 (선택 A/B/C) → 스탯 변화만 다름
  ↓ (합류)
스테이지3 (선택 A/B/C) → 스탯 변화만 다름
  ↓ (합류)
최종 관문 → 누적 스탯에 따라 진짜 분기
  ↓
엔딩 A (퍼펙트) / 엔딩 B (일반생존) / 엔딩 C (사망) / 히든
```

엔딩 조건 명세 (필수 — scenario.json에 포함):
- 퍼펙트 생존: 구체적 스탯 임계값 (예: 전체 스탯 평균 70 이상)
- 일반 생존: 구체적 스탯 임계값
- 사망: 특정 스탯 0 도달 또는 지뢰 3개 이상 적중
- 히든: 특수 트리거 (예: 특정 스테이지에서 아무것도 선택하지 않음)

scenario.json에 분기 트리 + 엔딩 조건까지 포함 후 잠금.

## 파트 2: 7파트 서사 개요

각 파트마다 아래 항목을 채운다:

```
[파트 1: 오프닝]
핵심 사건: VR 접속 + 낭만 파괴 + 신분 부여
감각 키워드: [청각] [촉각] [후각]
UI 멘트: "시스템 알림. [내용]..."
오르페아 포인트: "[냉소적 한마디]"

[파트 2: 매치업]
핵심 사건: 위기 인스턴스 활성화, 빌런 등장
감각 키워드: ...
심는 아이템/복선: ...

[파트 3: 스테이지 1]
함정: [팩트ID]
유혹적 선택: ...
배드 엔딩: [어떻게 죽는가]
탈출구: [어떻게 살아나는가]
획득 아이템: ...

[파트 4: 스테이지 2]
...

[파트 5: 스테이지 3]
...

[파트 6: 스테이지 4 / 최종관문]
심어둔 아이템 전부 회수
최종 폭로 방식: ...
배드 엔딩 반전: 갑작스러운 접속 끊김

[파트 7: 마무리]
역사적 사실 기반 에필로그
오르페아 의미심장한 축하
구독/좋아요 → 게임 별점으로 치환
```
