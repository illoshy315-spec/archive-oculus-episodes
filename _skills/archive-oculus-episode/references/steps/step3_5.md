# Step 3.5 — 서사 설계 + 복선 연결 + scenario.json 완성

## 임무
scenario.json 골격에 복선/아이템/인과관계를 추가하고
7파트 서사 개요를 완성한다.
이 단계 완료 후 scenario.json을 잠근다.

---

## 파트 1: 인과관계 설계

각 스테이지 간 연결을 명시한다.
"파트 X에서 심은 [아이템/정보/관계]가 파트 Y에서 어떻게 회수되는가"

| 심는 파트 | 심는 것 | 회수 파트 | 회수 방식 |
|-----------|---------|-----------|-----------|
| 파트2 | [아이템명] | 파트5 | [어떻게 씀] |
| 파트3 | [NPC 관계] | 파트6 | [어떻게 씀] |

---

## 파트 2: 7파트 서사 개요

각 파트마다 아래 항목을 채운다:

```
[파트 1: 오프닝]
핵심 사건: VR 접속 + 낭만 파괴 + 신분 부여
감각 키워드: [청각] [촉각] [후각]
UI 멘트: "시스템 알림. [내용]..."
오르페아 포인트: "[냉소적 한마디]"

[파트 2: 매치업]
핵심 사건: 위기 인스턴스 활성화, 빌런 등장
감각 키워드: ...
심는 아이템/복선: ...

[파트 3: 스테이지 1]
함정: [팩트ID]
유혹적 선택: ...
배드 엔딩: [어떻게 죽는가]
탈출구: [어떻게 살아나는가]
획득 아이템: ...

[파트 4: 스테이지 2]
...

[파트 5: 스테이지 3]
...

[파트 6: 스테이지 4 / 최종관문]
심어둔 아이템 전부 회수
최종 폭로 방식: ...
배드 엔딩 반전: 갑작스러운 접속 끊김

[파트 7: 마무리]
역사적 사실 기반 에필로그
오르페아 의미심장한 축하
구독/좋아요 → 게임 별점으로 치환
시청자 참여 질문
```

## 파트 3: scenario.json 최종 업데이트

파트 1에서 설계한 인과관계를 scenario.json stages[]의
`planted_item`과 `foreshadow` 필드에 채운다.

---

## 🔒 scenario.json 잠금

사용자 승인 후 scenario.json을 확정 잠금한다.
이후 Step 4 집필 중 scenario.json 내용은 **변경 불가**.
집필 중 변경이 필요하면 사용자에게 보고하고 재승인을 받는다.
