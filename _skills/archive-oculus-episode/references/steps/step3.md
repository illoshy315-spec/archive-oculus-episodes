# Step 3 — 설계 (포맷 → 파트 구조 → 지뢰 → 신분 → NPC → scenario.json)

## 임무
에피소드의 뼈대를 설계한다.
**순서가 중요하다 — 아래 순서를 반드시 지킬 것.**

---

## 순서 원칙

```
포맷 결정
    ↓
파트 구조 확정 (스테이지 수 결정)
    ↓
파트별 지뢰 설계 (파트당 2~3개 후보)
    ↓
파트당 1개 선택
    ↓
플레이어 신분 역산 (지뢰가 자연스럽게 등장하는 신분)
    ↓
고정 NPC 등장 여부 결정
    ↓
에피소드 NPC 카드 작성
    ↓
scenario.json 골격 생성
```

---

## 파트 1: 포맷 결정

`references/episode_formats.md` 참고.
포맷에 따라 스테이지 수와 Step 5 빌드 템플릿이 달라지므로 반드시 먼저 확정.

| 포맷 | 스테이지 수 | 특징 |
|------|------------|------|
| 기본 생존형 | 3개 | 기본 템플릿 |
| 로그라이크 | 2개 | 짧고 반복 |
| 육성 시뮬레이션 | 4~5개 | 날짜 단위 |
| 법정물 | 2개 | 심문 + 최종 변론 |
| 미연시 | 3개 | 호감도 연동 |
| 추리물 | 3개 | 단서 수집 |
| 탈출어드벤처 | 3개 | 아이템 조합 |
| 타이쿤 | 4개 | 자원 관리 |

→ 포맷 확정 후 사용자 승인.

---

## 파트 2: 파트 구조 확정

포맷 기준으로 에피소드의 파트 흐름을 먼저 잡는다.
각 파트에서 어떤 상황이 일어나는지 한 줄씩 서술.

```
[오프닝]    어떤 상황에서 시작하는가?
[스테이지1] 어떤 상황이 벌어지는가?
[스테이지2] 상황이 어떻게 악화되는가?
[스테이지N] ...
[최종관문]  어떤 극한 상황에 몰리는가?
[엔딩]      생존 / 사망 / 히든의 결과는?
```

→ 파트 구조 확정 후 사용자 승인.
→ 이 구조가 확정돼야 지뢰 설계로 넘어간다.

---

## 파트 3: 파트별 지뢰 설계

**파트 구조가 확정된 후에 설계한다.**
각 파트의 상황에 맞는 지뢰 후보를 2~3개씩 설계.
`references/choice_types.md` 참고 — 유형(역사적함정/인물관련/스탯돌발/아이템활용)을 파트마다 다르게 배분.

**사회/문화 맥락 필수 반영** — `references/social_cultural_context.md` 로드.
각 스테이지마다 아래 세 가지를 먼저 답한 후 선택지를 설계한다:
1. 이 파트에서 플레이어를 옥죄는 사회/문화 구조는 무엇인가?
2. 현대 상식으로 당연한 행동이 왜 당시엔 위험한가?
3. 선택지 중 어느 것이 그 시대 논리로 "맞는" 선택인가?

→ 파트별 후보 A/B/C = 그 스테이지의 실제 선택지 A/B/C.
   "하나를 고른다"가 아님 — 세 개가 전부 들어간다.

| 파트 | 상황 | 지뢰 후보 | 선택지 유형 | 흥미도 |
|------|------|-----------|------------|--------|
| 오프닝 | ... | 없음 (세계관 진입) | — | — |
| 스테이지1 | ... | 후보A / 후보B | 인물관련 or 역사적함정 | /10 |
| 스테이지2 | ... | 후보A / 후보B | 스탯돌발 or 역사적함정 | /10 |
| 스테이지N | ... | 후보A / 후보B | 아이템활용 or 역사적함정 | /10 |
| 최종관문 | ... | 후보A / 후보B | 역사적함정 집중 | /10 |

→ fact_sheet.md의 팩트를 지뢰 소재로 활용.
→ 파트당 후보 2~3개 제시 → 사용자가 파트당 1개 선택.

**유형 배분 권장 (기본 생존형 3스테이지 기준):**
```
오프닝    → 선택지 없음
스테이지1 → 인물 관련 (NPC 신뢰/배신 복선 심기)
스테이지2 → 스탯 돌발상황 or 아이템 획득
스테이지3 → 역사적 함정 + 아이템 활용
최종관문  → 역사적 함정 2개 (클라이맥스)
```

---

## 파트 4: 플레이어 신분 역산

**지뢰가 확정된 후에 결정한다.**

질문: "확정된 지뢰들이 한 에피소드 안에서 자연스럽게 발생하려면
      플레이어가 어떤 위치에 있어야 하는가?"

신분 후보 2~3개 제안 → 사용자 선택.
각 신분의 강점/약점/게임적 긴장감 비교 제시.

---

## 파트 5: 고정 NPC 등장 여부 결정

`references/fixed_npc_cards.md` 참고.
- 역병의사: 이번 에피소드에 등장하는가? 어떤 가면/방식으로?
- 네피아: 복선을 심을 단계인가? (오르페아 말투 변화 등)
- 글리치 NPC 후보: 에피소드 NPC 중 나중에 타락시킬 후보가 있는가?

→ 강제 아님. 등장 안 해도 무방.
→ EP.01 기준: 역병의사 실루엣만 등장 권장 (세계관 복선 최소).

---

## 파트 6: NPC 카드 작성

`references/npc_card_template.md` 형식으로 등장인물마다 작성.
최소 구성: 빌런1 / 보스(또는 빌런2) / 조력자 1명.

원칙:
- 달콤해 보이는 NPC = 함정
- 차가워 보이는 NPC = 진짜 조력자
- 각 NPC의 동기는 반드시 표면/실제 두 겹으로 설계

---

    }
  ],
