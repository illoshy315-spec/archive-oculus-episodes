# Step 2 — 딥 리서치 + Fact Sheet + 스탯 설계

## 임무
선택된 주제에 대해 researcher 에이전트를 실행하고
fact_sheet.md를 생성한 뒤 에피소드 스탯을 설계한다.

## 실행
`agents/researcher.md` 지시에 따라 레딧 5개 병렬 탐색.
fact_sheet.md 생성 후 사용자에게 보고.

## 추가: 에피소드 스탯 설계

fact_sheet.md 완성 직후 다음 질문으로 스탯 4~6개를 확정한다:

**질문 1**: 이 시대에서 생존을 결정짓는 핵심 요소 3~5가지는?
→ 각 요소가 스탯 후보

**질문 2**: 잘못된 선택으로 깎이는 스탯은? 올바른 선택으로 올라가는 스탯은?
→ 함정과 스탯 연결

**질문 3**: 서로 trade-off 관계인 스탯 조합이 있는가?
→ 게임적 긴장감의 핵심

### 스탯 설계 예시
```
Salem 1692:
  신앙 (100): 예배 참석, 기도로 상승 / 이성적 발언으로 하락
  평판 (100): 이웃 관계 / 마녀 의혹으로 급하락
  법적방어력 (100): 법률 지식 활용 / 무지하면 0
  정신력 (100): 극한 상황에서 소모 / 0이면 자백

WW2 노르망디:
  체력 (100): 부상으로 하락
  탄약 (50발): 전투마다 소모
  엄폐도 (100): 지형에 따라 변동
  부대신뢰도 (100): 전우 관계
```
