# Step 4 — 롱폼 대본 집필

## 집필 시작 전 필수 확인
1. scenario.json 🔒 잠금 확인
2. progress_log.md 로드 (없으면 `references/progress_log_template.md`로 생성)
3. `references/orphea_voice_guide.md` 로드
4. `references/harness.md` 로드

---

## 파트별 집필 규칙

### 파트 1: 오프닝
- VR 접속 메타포로 시작
- 시청자가 상상하는 낭만적 이미지 제시 (30초 분량)
- 즉각 파괴: 오감으로 가혹한 실체 투하
- 플레이어 신분/스탯/미션 부여
- 오르페아 시그니처 오프닝 포함

### 파트 2: 매치업
- 설명 없이 현장 직행 (In Media Res)
- 첫 문장부터 위협 또는 돌발 사건으로 시작
- 빌런/환경적 위기 3~4개 유기적으로 연결
- 오르페아가 플레이어의 현대적 상식을 비웃음
- UI 동기화 멘트 포함

### 파트 3~6: 스테이지
각 스테이지마다:
1. scenario.json 해당 스테이지 팩트/함정 확인
2. 달콤한 선택 유도
3. 배드 엔딩 시나리오 (팩트 기반 파멸 폭로)
4. 정답 선택으로 탈출
5. 선택지 포맷 삽입 (harness.md 규정 형식)
6. 획득/회수 아이템 처리
7. 다음 스테이지 연결

### 파트 7: 마무리
- 역사적 사실 기반 에필로그
- 오르페아 의미심장한 축하 (최대 감정 표현)
- 시청자 참여 질문
- 구독/좋아요 → 게임 별점으로 치환

---

## 파트 완성 후 필수 체크

파트 제출 전 반드시 확인:
```
□ 분량: 1000자 이상인가?
□ 청각 묘사: 2개 이상인가?
□ 촉각/후각 묘사: 2개 이상인가?
□ NPC 대사(" "): 3문장당 1회 이상인가?
□ 오르페아: 설명이 아닌 독설로 말하고 있는가?
□ 오르페아 금지어 없는가? (여러분/우리/화이팅 등)
□ scenario.json의 해당 스테이지 팩트를 사용했는가?
□ progress_log.md 업데이트했는가?
□ 연출 가이드/레이블이 포함되지 않았는가?
```

하나라도 미달이면 수정 후 제출.
