## VN 스크립트 집필 원칙

### 대사
- 한 텍스트박스 = 2~3줄 이내
- NPC 대사는 캐릭터의 목적/욕망이 드러나게
- 오르페아: 씬당 최대 2회 등장, 항상 짧고 냉소적

### 플레이어
- 행동은 선택지(CHOICE)로만 표현
- 지문으로 플레이어 행동을 서술하지 않는다
- 선택지 텍스트는 1줄 이내

### 스탯/아이템
- 대사 안에 섞지 않는다
- `STAT:` / `ITEM_GET:` / `ITEM_USE:` 태그로 분리

---

## 씬 구성 순서 (스테이지당)

```
1. 배경 씬 (BG + BGM 설정)
2. 상황 도입 NPC 대사 1~2줄
3. 오르페아 코멘트 (선택적)
4. 갈등 고조 NPC 대사 2~3줄
5. 스탯 변화 알림
6. 오르페아 선택지 직전 평가
7. CHOICE → 분기
   - 나쁜 선택: BAD_END 씬으로
   - 좋은 선택: 다음 스테이지로
```

---

## BAD_END 씬 필수 요소
```
BG: 파멸 배경 (교수대, 감옥, 전쟁터 등)
NARR: 실제 역사적 결과 한 줄 (팩트 기반)
ORPHEA: 마무리 냉소 한 마디
GOTO: game_over
```

---

## 씬 완성 후 필수 체크

```
□ 대사가 텍스트박스 2~3줄 이내인가?
□ 오르페아가 씬당 2회 이하인가?
□ 선택지가 명확하게 갈림길을 제시하는가?
□ 스탯 변화가 STAT: 태그로 분리되었는가?
□ BAD_END에 실제 역사 팩트가 포함되었는가?
□ scenario.json의 해당 스테이지 trap_fact를 사용했는가?
□ 오르페아 금지어 없는가?
□ progress_log.md 업데이트했는가?
```

하나라도 미달이면 수정 후 다음 씬.
