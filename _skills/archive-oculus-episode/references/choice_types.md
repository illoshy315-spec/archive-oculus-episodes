# 선택지 유형 분류 가이드

Archive Oculus 모든 에피소드에 적용되는 선택지 설계 기준.
시나리오 작성 시 각 파트마다 유형을 의식적으로 배분할 것.

---

## 선택지 개수: 3개 고정

```
A: 현대 상식 함정 (또는 감정적으로 끌리는 선택)
B: 안전하지만 대가가 따르는 선택
C: 리스크 있지만 보상이 있는 선택
```

매 스테이지마다 이 구조를 유지. 유형만 바뀜.

---

## 유형 1 — 역사적 함정 (Historical Trap)

**정의**: 현대 상식으로는 정답처럼 보이지만, 당시 시대 논리로는 치명적인 선택.
Step 2 리서치에서 설계한 지뢰가 이 유형에 해당.

**특징**:
- 플레이어가 틀렸다는 걸 몰라야 함 — 알면 함정이 아님
- 틀린 후 오르페아의 "그럴 줄 알았습니다"가 가장 살아있는 순간
- 에피소드당 3~4개. 클라이맥스(법정/최종관문)에 집중 배치 가능

**세일럼 예시**:
- "저는 결백합니다" → 결백 주장한 19명 전원 교수형
- 고발자 소녀가 연기한다고 반박 → 본인이 즉시 고발 대상
- 법정에 토지 서류 제출 → 재산 규모 노출, 재판관 표적

---

## 유형 2 — 인물 관련 (NPC Relationship)

**정의**: NPC와의 신뢰/배신/협력을 결정하는 선택.
즉각 결과가 아닌 **이후 파트에서 결과가 돌아오는** 지연 효과.

**특징**:
- 호감도 시스템과 연동 (scenario.json NPC 호감도 필드)
- 잘못된 선택 → 이후 파트에서 해당 NPC가 배신 or 외면
- 올바른 선택 → 이후 파트에서 조력자로 등장
- 달콤해 보이는 NPC가 함정, 차가워 보이는 NPC가 진짜 조력자 원칙

**예시 패턴**:
```
파트 2: 이웃 A가 도움을 제안함 → 믿을 것인가?
  → 믿으면: 파트 5에서 A가 공범으로 지목함 (배신)
  → 거절하면: A가 섭섭해하지만 파트 5에서 침묵 (중립)

파트 3: 같은 감옥 수감자 B가 탈출 방법을 알려줌 → 따를 것인가?
  → 따르면: B가 실제로 도움 (조력자)
  → 무시하면: 탈출 기회 소실
```

**세일럼 예시**:
- 아비게일이 "당신만은 고발하지 않겠다"고 접근 → 믿으면 파트 5에서 배신
- 감옥 동료가 자백 노하우를 귀띔 → 차갑게 보이지만 진짜 조력자
- 재판관 코윈이 "협조하면 선처" 제안 → 믿으면 재산 몰수 당함

## 유형 3 — 스탯 돌발상황 (Stat Emergency)

**정의**: 특정 스탯이 임계값 이하로 내려가면 강제 발동되는 선택지.
플레이어가 예상하지 못한 시점에 등장하는 것이 핵심.

**특징**:
- 선택지가 아니라 "상황"이 먼저 와야 함 — 스탯 감소 → 사건 발생 → 선택 강요
- 역사적 사건/질병/환경을 트리거로 활용
- 스탯 관리를 잘못한 플레이어에게 패널티 선택지 부여
- 스탯 관리를 잘한 플레이어는 더 나은 선택지가 열림

**트리거 구조**:
```
[스탯 X] ≤ [임계값] → [사건 발생] → [선택지 A/B/C]
```

**세일럼 예시**:
```
정신력 ≤ 30:
  트리거: 맥각균에 감염된 빵을 먹음 (호밀빵, 당시 흔한 식량)
  사건: 환각 증상 발생. 법정 안에서 이상한 소리가 들리기 시작
  선택:
    A. 증상을 숨긴다 → 정신력 -10, 하지만 의심 안 받음
    B. 쓰러진다 → 고발자들과 같은 증상 → 마녀 의심 즉시 해제되지만 체면 -평판
    C. "저도 악마를 봤습니다" 자백 → 신앙 +20, 생존율 상승, 하지만 공범 지목 압박

신앙 ≤ 20:
  트리거: 예배에 오래 빠짐
  사건: 목사가 설교 중 플레이어를 공개적으로 언급
  선택:
    A. 즉시 교회로 달려간다 → 평판 회복, 체력 -
    B. 무시한다 → 평판 급감
    C. 아픈 척한다 → 중간 결과
```

**다른 에피소드 활용 예시**:
- 흑사병 런던: 체력 ≤ 20 → 발열 증상 → 격리 or 숨길 것인가
- 빅토리아 하녀: 체력 ≤ 30 → 과로 쓰러짐 → 해고 위기

## 유형 4 — 아이템/정보 활용 (Item/Info Use)

**정의**: 앞 파트에서 획득한 아이템이나 정보를 적절한 타이밍에 사용하는 선택.
"준비한 플레이어에게 탈출구가 열린다"는 원칙.

**특징**:
- 아이템/정보는 반드시 파트 2(일상)에서 심어야 함
- 회수는 파트 5~6(클라이맥스)에서
- 아이템 없으면 해당 선택지가 회색(비활성)으로 표시
- 과도한 힌트 없이 "가져왔다면 쓸 수 있다"는 식으로 자연스럽게

**구조**:
```
파트 2에서 심기:
  [아이템/정보명] 획득
  → scenario.json planted_item 필드에 기록

파트 5~6에서 회수:
  CHOICE:
    A. [아이템] 제시 → 탈출구 (아이템 보유 시만 활성)
    B. 말로 설득한다 → 낮은 성공률
    C. 포기한다 → 스탯 손실
```

**세일럼 예시**:
```
파트 2에서 심기:
  - 토지 양도 서류 (재산을 미리 아들에게 넘기는 문서)
  - 재판관 코윈의 이해충돌 증거 (그가 플레이어 땅을 원한다는 정보)
  - 감옥 동료에게 들은 "자백 후 공범 지목 최소화" 노하우

파트 6 법정에서 회수:
  코윈이 재산 양도 서류 요구 →
    A. [토지 양도 서류 있음] "이미 처리했습니다" → 몰수 불가, 코윈 무력화
    A'. [서류 없음] 회색 처리 → 선택 불가
    B. "고려해보겠습니다" → 시간 벌기, 스탯 소모
    C. 거부한다 → 즉시 유죄 판결 압박
```

## 파트별 유형 배분 가이드 (기본 생존형)

```
파트 1  오프닝        → 선택지 없음 (세계관 진입)
파트 2  일상          → 인물 관련 + 아이템 획득
파트 3  위기 발생     → 역사적 함정 1
파트 4  상황 악화     → 스탯 돌발상황 or 인물 관련
파트 5  탈출 시도     → 역사적 함정 2 + 아이템 활용
파트 6  최종 관문     → 역사적 함정 3~4 (클라이맥스)
파트 7  엔딩          → 선택지 없음 (결과 전달)
```

```json
{
  "choice_type": "historical_trap | npc_relation | stat_emergency | item_use",
  "npc_affection_change": {"npc_id": "abigail", "delta": -20},
  "stat_trigger": {"stat": "sanity", "threshold": 30},
  "required_item": "land_deed",
  "planted_in_stage": 2
}
```
