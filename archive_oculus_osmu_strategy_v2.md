## 1. 채널별 콘텐츠 형태 (확정)

```
유튜브 (롱폼)
게임 플레이 영상 편집본 + 나레이션, 전체 스토리 공개

틱톡/숏츠
분기점(선택의 순간) 위주 편집
⚠️ 클리프행어 원칙: 선택 직전 ~ 오르페아 코멘트까지만 보여주고
   결과는 보여주지 않음 ("당신이라면? 👉 게임에서 확인")
   결과까지 보여주면 게임 유입 동기가 사라짐

인스타/스레드 (카드뉴스)
역사적 맥락/상황 설명. 다크 아카데미아 비주얼 큐레이션
정보량: fact_sheet.md의 일부만 (자극적 핵심만, 휘발성)
관점: 1인칭 몰입형 ("당신이 그 시대에 있었다면")
마지막 장: 게임 유입 CTA + 오르페아 캐릭터 노출 (떡밥 겸용)

웹 — 커뮤니티 + 아티클 (역할 변경)
단순 블로그가 아니라 토론/제보가 쌓이는 공간으로 설계
세부는 4번 항목 참고
```

## 2. 콘텐츠 중복 방지 — "채택되지 않은 기록" 전략

### 문제
카드뉴스와 웹 아티클이 같은 fact_sheet.md를 쓰면 콘텐츠가 겹침
(알고리즘 중복 패널티 + 유저가 "또 같은 내용"이라고 느낌)

### 해결책 — 정보량/관점 차등화 + 새로운 콘텐츠 축 추가

```
1차 해법: 가공 방식 차별화
카드뉴스 = 맛보기 + 임팩트 (fact_sheet.md의 일부)
웹 아티클 = 정찬 + 깊이 (fact_sheet.md 전체 + 추가 분석)

2차 해법 (★ 더 강력함): "채택되지 않은 기록" 콘텐츠 축 신설
Step 3에서 지뢰 후보 6개 중 4개만 채택하고 2개는 버려짐
→ 이 버려진 2개가 지금까지 콘텐츠로 전혀 안 쓰이고 있었음
→ 이걸 웹 아티클 전용 콘텐츠로 활용

형식: "세일럼 마녀재판: 게임에 담지 못한 기록들"
내용: 지뢰로 채택 안 된 역사적 사실 1개당 1섹션
      - 왜 이게 지뢰가 될 수 있었는지
      - 당시엔 왜 이런 일이 가능했는지 (사회/문화적 맥락)
      - "만약 게임에 있었다면 어떤 선택지였을까" (재미 요소)

효과:
게임/카드뉴스/숏츠 = "채택된 이야기" (압축, 핵심)
웹 아티클 = "채택되지 않은 이야기" (확장, 깊이)
→ 같은 소스에서 나왔지만 완전히 다른 콘텐츠
→ 게임 해본 사람도 "오 이것도 있었어?" 새로운 정보로 느낌
```

## 3. 클코 파이프라인 연동 — Step 2/3 산출물 추가 활용

```
Step 2 완료 시점 (게임 제작 시작 전)
→ fact_sheet.md 완성 직후 카드뉴스 제작 착수 가능
→ 게임 만드는 기간 동안 SNS는 미리 가동 (마케팅 공백 없음)

→ 채택 안 된 지뢰 후보 2개를
  "채택되지 않은 기록" 웹 아티클 재료로 자동 보관
→ 클코 워크플로우에 "미채택 지뢰 보관" 단계 추가 권장
```

## 4. 웹 = 커뮤니티 설계

### 콘텐츠 엔진은 이미 세계관에 내장되어 있음

```
ARG 복선 시스템      → "이거 발견했는데 너네도 봤어?" 제보 스레드
오르페아 vs 네피아   → 진영 논쟁 스레드 자연 발생
역병의사 정체        → 추리/미스터리 게시판
글리치 NPC 목격담    → "나도 3화에서 봤어" 제보형 재방문 유도
채택되지 않은 기록   → "만약 이것도 지뢰였다면?" 유저 참여형 가정 스레드
```

### 플랫폼 선택

```
초반: 기존 플랫폼 활용 (디스코드 + 레딧 서브레딧 등)
      → 진입장벽 낮음, 초기 트래픽 모으기 쉬움
팬덤 형성 후: 자체 웹 커뮤니티로 이전/병행
      → 브랜드 완전 소유
```

### 첫 떡밥 설계 원칙

```
EP.01부터 "토론거리 1개"를 의도적으로 명확히 심을 것
너무 어렵지도 너무 쉽지도 않은 난이도
예: "역병의사가 왜 거기 있었을까?"
```

## 5. 브랜드 보이스 — 오르페아 페르소나 (전 채널 통일, 확정)

### 원칙

```
"Archive Oculus 공식" 계정이 오르페아 캐릭터 보이스로 운영
(브랜드 페르소나 마케팅 기법 — 공식임을 명시한 채널에서만 적용)

⚠️ 절대 금지: 기존 커뮤니티(레딧 등)에 정체를 숨기고
   오르페아인 척 잠입 활동하는 것 — 자작극(astroturfing)에 해당,
   발각 시 브랜드 신뢰도에 치명타. 공식 채널에서 투명하게 운영할 것.
```

### 적용 범위

```
유튜브 — 영상 나레이션
인스타/스레드 — 카드뉴스 캡션, 댓글 반응
디스코드/커뮤니티 — 공지, 운영 멘트
웹 아티클 — 본문 톤

전 채널 동일한 오르페아 보이스로 일관성 유지
→ 어느 채널에서 만나도 "같은 세계"라는 느낌
```

### 보이스 예시

```
카드뉴스 캡션:
"1692년, 세일럼. 의심받는 데 이유는 필요 없었습니다. —기록 보관자, 오르페아"

댓글 반응:
"흥미로운 질문이군요. 답은... 기록되어 있습니다. 직접 찾아보시길."

업데이트 공지:
"새로운 기록이 추가되었습니다. 이번엔 누가 살아남을까요. 유감스럽게도, 기대는 안 합니다."

디스코드 환영 메시지:
"또 한 명의 관찰자가 접속했군요. 기록을 즐기시길. 살아남는 건 당신 몫입니다."
```

### 예외 — CS/실무 응대는 톤 분리 (확정: 네피아 담당)

```
오르페아 보이스는 "콘텐츠성 소통"에만 적용
(공지, 캡션, 가벼운 댓글, 떡밥)

실무성 소통(버그 신고, 결제 문의, 진지한 불만)은
네피아 보이스로 전환 — 친절하되 기대 없는 톤으로 명확하게 처리
→ 캐릭터 일관성 유지하면서도 실무 응대 명확성 확보
→ 부가 효과: "버그 신고하면 네피아가 받는다"는 것 자체가
   재미 요소로 작동할 수 있음 (단, 실제 응대 품질은 진지하게 유지)
```

## 6. 숏츠 편집 원칙 (확정 — 2종 + 공통 규칙)

### 숏츠 2종 배치

```
타입 A — 클리프행어형 (롱폼 공개 "전" 게시)
포함: 선택지 제시 직전까지의 긴장감 있는 장면
포함: 선택 직후 오르페아의 즉각 반응/코멘트
제외: 선택의 결과 (스탯 변화, 이후 전개)
끊는 지점: 오르페아 코멘트 직후, 결과 공개 전
텍스트 오버레이: "당신이라면? 👉 게임에서 확인" 류 CTA
목적: 기대감 조성, 롱폼 공개 시점 대기 수요 형성
주요 반응 지표: 저장 (나중에 결말 보려고)

타입 B — 배드플레이형 (롱폼 공개 "후" 게시)
일부러 잘못된 선택지를 고르는 과정을 보여줌
오르페아의 냉소적 코멘트 + 배드엔딩 진입 직전까지 공개
"진짜 살아남는 법"(정답 루트)은 숨김 — 게임에서 찾도록 유도
목적: 유머/우월감 자극("나라면 안 그랬다"), 자발적 공유 유도
주요 반응 지표: 공유 (친구에게 추천하는 행위)
코미디 지뢰 전용 소재로 활용 (game_pipeline_v1.md Step 3 참고)

이유: 결과를 보여주는 타이밍과 톤이 서로 다른 목적을 가짐
      A=궁금증으로 유입, B=재미로 공유 → 같은 에피소드로 두 효과 모두 노림
```

### 숏츠 제작 공통 원칙 (2026 알고리즘 리서치 반영)

```
1~3초 내 강력한 훅 필수
텍스트오버레이 + 사운드로 즉각적 임팩트
(기존 GPT 로드맵 "1.5초 법칙"과 일치 — 강화 확정)

반복 가능한 고정 포맷 사용
매 숏츠 같은 틀 유지: [상황 제시] → [선택지] → [오르페아 코멘트] → [CTA]
→ 시청자가 "이거 그 시리즈다" 즉시 인식하게 함
→ 채널 정체성 반복 시그널 (2026 트렌드 권장사항)

성공 지표 우선순위: 좋아요 < 공유/저장
목표: 조회수 대비 참여율 6% 이상 (2026년 유튜브 쇼츠 기준)
저장 = 클리프행어형이 강함 / 공유 = 배드플레이형이 강함

기술 요건
세로형 포맷 필수
명확한 캡션(자막) 필수 — 무음 시청 대응
```

### 코미디 지뢰 연동

```
game_pipeline_v1.md Step 3에서 설계되는 "코미디 지뢰"가
타입 B(배드플레이형) 숏츠의 핵심 소재가 됨

코미디 지뢰 조건 (Step 3 참고):
- 실제 역사적 사실 기반 (허구 금지, 고증 신뢰도 보호)
- 출처 URL 명시 가능한 것만 채택
- 적합한 실제 사례가 없으면 과감히 패스 (강제 아님)
  → 그 경우 타입 B 숏츠는 일반 배드엔딩 루트로 대체 제작
- 오르페아는 웃기려 하지 않으나 결과적으로 웃긴 블랙코미디 톤
- 에피소드당 1~2곳에만 배치
```

## 9. 유튜브 알고리즘 대응 전략 (2026 기준 리서치 반영)

### 알고리즘 작동 방식 — 3단계 마이크로 테스트

```
새 영상은 한 번에 크게 노출되지 않고 단계적으로 테스트됨

1단계 (핵심 시청자): 구독자 + 정기 시청자에게 우선 노출
2단계 (잠재 시청자): 1단계 통과 시, 비슷한 주제를 보는 비구독 신규 시청자에게 노출
3단계 (신규 대규모 시청자): 2단계까지 통과해야 완전 신규 대중에게 추천 시작

평가 기준: 클릭률, 시청 지속 시간, 실제 참여도(좋아요/댓글/공유)
98% 이상의 영상이 3단계 통과 실패 — 매우 치열한 경쟁

⚠️ 우리 전략에 주는 함의:
EP.01의 "1단계 통과 여부"는 초기 노출 대상의 반응에 달려있음
→ 섹션 7(공개 순서)의 "SNS 먼저 가동 → 핵심 시청자 풀 확보"가
   1단계 테스트 통과율을 높이는 핵심 장치임을 재확인
→ EP.01 공개 전 SNS/커뮤니티에 일정 규모의 대기 인원을
   만들어두는 것이 알고리즘 통과에 직결됨
```

```

→ 도입부 30초 안에 "왜 끝까지 봐야 하는지" 명확히 전달 필요

   (기존 GPT 로드맵의 "POV: 당신은 1351년 흑사병..." 훅 공식과 일치)

이탈 지점(drop-off) 분석: 어느 지점에서 시청자가 이탈하는지 추적
→ EP.01 공개 후 이탈 구간 데이터를 보고 EP.02 구조 조정에 반영
```
